# F8.3 — Editor de Normalización (reglas de limpieza previa al matching)

## Objetivo

Activar el ítem **Normalización** del grupo "Clasificación y aprendizaje · admin" con un
editor admin sobre `reglasNormalizacion`: las reglas ordenadas (`prefix`/`suffix`/
`replace`/`regex`) que limpian el texto **antes** de que el clasificador/matcher busquen
patrón. Es el editor más delicado del set: el orden importa y un regex inválido corrompe
la clasificación de todo lo que entra.

Esta fase entrega, con dos salvavidas:
- **Preview en vivo** (client-side puro): ves el texto de muestra transformado **regla por
  regla en orden** antes de guardar.
- **Validación server-side de regex** en la callable: un patrón que no compila se rechaza,
  nunca se persiste.

`reglasNormalizacion` ya tiene `read: esMiembro` (el clasificador la lee) y `write:false`
→ **no hay cambio de rules**. Mutación por callable.

---

## Contexto verificado (origin HEAD `df090fd`)

**Shape de `reglasNormalizacion/{id}`** (del seed `normalizationRules.ts`):
```
id       : 'rule_000' (seed) — las nuevas usan auto-id
tipo     : 'prefix' | 'suffix' | 'replace' | 'regex'
patron   : string
reemplazo: string            // puede ser '' (válido)
activo   : boolean
orden    : number            // define el orden de aplicación
notas    : string | null
```

**⚠ Trampa `activo`:** hoy **ningún lector filtra por `activo`**:
- Server `cargarReglasNormalizacion` (`functions/src/index.ts`): `.sort(orden)` sin filtro.
- Cliente `DiccionarioContext.tsx`: `getDocs(collection('reglasNormalizacion'))` sin `where`.

→ Si el editor expone un toggle `activo` sin tocar los lectores, sería **no-op silencioso**
(mismo problema que el `activo` de destinos). Este prompt lo hace **real** filtrando en
ambos lectores. Es un cambio a la entrada del pipeline → tiene test de regresión dedicado.
Como todos los docs sembrados tienen `activo:true`, **no hay cambio de comportamiento sobre
datos actuales**; solo habilita el soft-disable para reglas nuevas/editadas.

**Función pura de normalización (para el preview):** `normalizar(s, rules)` en
`src/datos/normalizador.ts` — espejo exacto del algoritmo server (`prefix`/`suffix`/
`replace`/`regex` con `try/catch` en regex inválido). El editor la usa **en el browser**,
sin backend, para el preview.

**Patrón de callable a espejar:** `guardarTaxonomia` (accion crear/editar/eliminar + auth
admin). **Twin-logic debt** conocida: `normalizar` está duplicada cliente/server; no se
suma copia nueva acá.

### 1b. Callable `guardarReglaNormalizacion` (crear / editar / eliminar / reordenar)

Espejar auth de `guardarTaxonomia`. Validar `tipo`, `patron` no vacío, y **compilar el
regex** cuando `tipo==='regex'`.

```ts
export const guardarReglaNormalizacion = onCall(
  { region: 'southamerica-east1' },
  async (request) => {
    if (!request.auth) throw new HttpsError('unauthenticated', 'No autenticado');
    const email = request.auth.token.email?.toLowerCase();
    if (!email) throw new HttpsError('unauthenticated', 'Email no disponible');
    const autSnap = await db.collection('autorizados').doc(email).get();
    if (!autSnap.exists || autSnap.data()?.rol !== 'admin') {
      throw new HttpsError('permission-denied', 'Se requiere rol admin');
    }

    const { accion } = (request.data ?? {}) as { accion?: string };
    if (!['crear', 'editar', 'eliminar', 'reordenar'].includes(accion ?? '')) {
      throw new HttpsError('invalid-argument', 'accion inválida');
    }
    const col = db.collection('reglasNormalizacion');

    // ── reordenar: ids en orden → orden = índice ──
    if (accion === 'reordenar') {
      const { ids } = request.data as { ids?: string[] };
      if (!Array.isArray(ids) || ids.length === 0) throw new HttpsError('invalid-argument', 'ids requerido');
      const batch = db.batch();
      ids.forEach((id, i) => batch.set(col.doc(id), { orden: i }, { merge: true }));
      await batch.commit();
      console.log(`[guardarReglaNormalizacion] reordenar ${ids.length} (por ${email})`);
      return { ok: true };
    }

    // ── eliminar ──
    if (accion === 'eliminar') {
      const { id } = request.data as { id?: string };
      if (!id) throw new HttpsError('invalid-argument', 'id requerido');
      await col.doc(id).delete();
      console.log(`[guardarReglaNormalizacion] eliminar ${id} (por ${email})`);
      return { ok: true };
    }

    // ── crear / editar: validar payload ──
    const { id, tipo, patron, reemplazo, activo, notas } = request.data as {
      id?: string; tipo?: string; patron?: string; reemplazo?: string;
      activo?: boolean; notas?: string | null;
    };
    if (!['prefix', 'suffix', 'replace', 'regex'].includes(tipo ?? '')) {
      throw new HttpsError('invalid-argument', 'tipo inválido');
    }
    if (typeof patron !== 'string' || patron.length === 0) {
      throw new HttpsError('invalid-argument', 'patron requerido');
    }
    if (tipo === 'regex') {
      try { new RegExp(patron, 'gi'); }
      catch (e) { throw new HttpsError('invalid-argument', `regex inválido: ${(e as Error).message}`); }
    }

    const payload: Record<string, unknown> = {
      tipo,
      patron,
      reemplazo: typeof reemplazo === 'string' ? reemplazo : '',
      activo: activo !== false,
      notas: notas ?? null,
    };

    if (accion === 'editar') {
      if (!id) throw new HttpsError('invalid-argument', 'id requerido para editar');
      const snap = await col.doc(id).get();
      if (!snap.exists) throw new HttpsError('not-found', 'Regla no encontrada');
      await col.doc(id).set(payload, { merge: true }); // no toca `orden`
      console.log(`[guardarReglaNormalizacion] editar ${id} (por ${email})`);
      return { ok: true, id };
    }

    // crear → orden al final
    const all = await col.get();
    const maxOrden = all.docs.reduce((m, d) => Math.max(m, (d.data().orden ?? 0) as number), -1);
    const ref = await col.add({ ...payload, orden: maxOrden + 1 });
    console.log(`[guardarReglaNormalizacion] crear ${ref.id} orden=${maxOrden + 1} (por ${email})`);
    return { ok: true, id: ref.id };
  },
);
```

## Cambio 3 — Wrapper `src/datos/reglasNormalizacion.ts` (NUEVO)

```ts
import { collection, getDocs } from 'firebase/firestore';
import { httpsCallable } from 'firebase/functions';
import { db, functions } from '../firebase';
import type { NormRule } from './normalizador';

export interface ReglaDoc {
  id: string;
  tipo: NormRule['tipo'];
  patron: string;
  reemplazo: string;
  activo: boolean;
  orden: number;
  notas: string | null;
}

export async function listarReglas(): Promise<ReglaDoc[]> {
  const snap = await getDocs(collection(db, 'reglasNormalizacion'));
  return snap.docs
    .map(d => {
      const x = d.data();
      return {
        id: d.id,
        tipo: x.tipo,
        patron: x.patron ?? '',
        reemplazo: x.reemplazo ?? '',
        activo: x.activo !== false,
        orden: typeof x.orden === 'number' ? x.orden : 0,
        notas: x.notas ?? null,
      } satisfies ReglaDoc;
    })
    .sort((a, b) => a.orden - b.orden);
}

export interface GuardarReglaInput {
  accion: 'crear' | 'editar' | 'eliminar' | 'reordenar';
  id?: string;
  ids?: string[];
  tipo?: NormRule['tipo'];
  patron?: string;
  reemplazo?: string;
  activo?: boolean;
  notas?: string | null;
}

export async function guardarRegla(input: GuardarReglaInput): Promise<{ ok: boolean; id?: string }> {
  const fn = httpsCallable(functions, 'guardarReglaNormalizacion');
  const res = await fn(input);
  return res.data as { ok: boolean; id?: string };
}
```

## Cambio 4 — Editor `src/vistas/perfil/Normalizacion.tsx` (NUEVO)

Estética espejando `Diccionario.tsx`. Spec con el **preview como pieza central**:

**Lectura**
- `listarReglas()` al montar (incluye inactivas, ya ordenadas por `orden`). Refetch tras
  cada mutación.

**Listado (ordenado por `orden`)**
- Cada fila: nº de orden, chip `tipo`, `patron` (mono) → `reemplazo` (mono, o "∅" si vacío).
- Filas `activo:false` atenuadas.
- Acciones por fila: subir/bajar (⬆⬇ → `guardarRegla({accion:'reordenar', ids:[...]})`
  con el array reordenado), toggle `activo` (`editar` con activo flip), `pencil`, `trash-2`.
- Botón "Nueva regla".

**Preview en vivo (panel fijo arriba o sticky)**
- Input de texto de muestra (default sugerido: `"COMPRA MERCADOPAGO*KIOSCO 12/03"`).
- Importar `normalizar` de `../../datos/normalizador`.
- Construir `reglasEfectivas` = reglas **activas** en orden, mapeadas a `NormRule`
  (`{tipo,patron,reemplazo}`). Cuando el bottom-sheet de edición/alta está abierto,
  **splicear la regla en edición** en su posición para previsualizar el cambio sin guardar.
- Mostrar transformación **paso a paso**: para cada regla activa en orden, el string
  resultante tras aplicarla (llamar `normalizar(entrada, reglasEfectivas.slice(0, i+1))`).
  Resaltar la fila de la regla en edición. Esto es lo que hace visible dónde una regla
  rompe el texto.
- Resultado final destacado abajo.
- Si `tipo==='regex'` y el patrón no compila, `normalizar` ya lo ignora (try/catch); mostrar
  un aviso inline "regex inválido — se ignora" para que no parezca que "no hace nada".

**Alta / edición (bottom-sheet)**
- Campos: `tipo` (segmented prefix/suffix/replace/regex), `patron` (mono, requerido),
  `reemplazo` (mono, permite vacío), `notas` (opcional), `activo` (switch, default on).
- El preview reacciona en vivo a lo tipeado.
- Submit → `guardarRegla({accion: id?'editar':'crear', ...})`.
- Manejar `HttpsError` (regex inválido) mostrando `message` bajo el campo `patron`.

**Aviso**: helper text "Los cambios impactan la clasificación tras recargar la app
(el diccionario se hidrata on-boot)."

## Regression / smoke tests (post-deploy)

1. **Visibilidad:** admin ve `/perfil/normalizacion` con las reglas reales ordenadas por
   `orden`; dependiente no accede.
2. **Preview:** con texto de muestra, el panel muestra la transformación paso a paso y el
   resultado final. Editar un patrón en el sheet cambia el preview **sin guardar**.
3. **Crear (replace):** nueva regla `replace` `patron:"*"` `reemplazo:" "` → guardar →
   aparece al final del orden. Verificar en Firestore: `activo:true`, `orden` = max+1.
4. **Regex inválido rechazado:** crear/editar `tipo:regex` con `patron:"("` (no compila) →
   callable rechaza `invalid-argument`, el sheet muestra el mensaje, **no** se persiste.
5. **Reordenar:** subir/bajar una regla → el orden persiste (recargar y sigue igual).
   Verificar campos `orden` reescritos 0..n.
6. **Soft-disable REAL (test clave):** desactivar una regla (`activo:false`). Recargar la
   app. Verificar en un texto que esa regla afectaba, que **ya no** se aplica (ni en el
   clasificador cliente ni — si aplica — en el matcher server). Reactivarla → vuelve a
   aplicarse. Confirma que el filtro `activo !== false` quedó en **ambos** lectores.
7. **No regresión de clasificación:** un comprobante/alta cuyo texto dependía de las reglas
   sembradas (todas `activo:true`) clasifica igual que antes de F8.3 (el filtro no cambió
   datos existentes).
8. **Eliminar:** borrar la regla de prueba (confirm) → desaparece; recargar confirma.
9. **Resto del grupo:** Diccionario (F8.1) y Destinos (F8.2) intactos.
