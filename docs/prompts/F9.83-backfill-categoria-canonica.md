# F9.83 — Backfill: asignar categoría canónica a movimientos que solo tienen subcategoría

> Script de migración en `scripts/seed/` sobre `movimientos` (patrón backfill F9.24: dry-run → `--apply`; lo corre el usuario).
> Fuente de verdad: la taxonomía en `config/familia` (subcategoría → `categoriaPadre`, F9.38).

---

## Por qué

Quedaron movimientos con `subcategoria` seteada pero **`categoria` vacía/null** (o inconsistente). Como los agregados
del Dashboard y "Por categoría" se derivan de `movimientos`, esos gastos caen fuera de su categoría (o en "sin categoría").
Hay que **completar `categoria` desde la subcategoría**, usando el mapa canónico de la taxonomía.

## §1 · Mapa subcategoría → categoría (de la taxonomía)

Construir el índice desde `config/familia`:

```ts
// subcategorias: [{ id/nombre, categoriaPadre, activo }]  (F9.38)
const subToCat = new Map<string, string>();
for (const s of (fam.subcategorias ?? [])) {
  const nombreSub = norm(s.nombre ?? s.id);
  const padre = s.categoriaPadre;               // nombre/label de la categoría canónica
  if (nombreSub && padre) subToCat.set(nombreSub, padre);
}
const norm = (s?: string) => (s ?? '').normalize('NFD').replace(/[\u0300-\u036f]/g,'').trim().toLowerCase();
```

## §2 · Selección y acción

- **Candidatos**: `movimientos` donde `subcategoria` no vacía **y** (`categoria` null/'' **o** `categoria` no está en
  la lista de categorías canónicas activas) **y** existe `subToCat.get(norm(subcategoria))`.
- **Acción**: `categoria = subToCat.get(norm(subcategoria))` + `actualizadoEn = serverTimestamp()`. No tocar nada más.
- **Sin match** (subcategoría que no está en la taxonomía, o movimiento sin subcategoría): **NO** inventar — loguearlos
  aparte como "revisión manual" (listado con `id`, `descripcion`, `subcategoria`) para decidir a mano.
- Dry-run por defecto (lista `id` · `subcategoria` · `categoria` actual → nueva + total). `--apply` escribe en batches ≤450.

## §3 · Reporte del dry-run (para vos)

Que imprima tres grupos:
1. **A completar** (tienen subcat con padre canónico) — los que va a escribir.
2. **Revisión manual** (subcat sin match en taxonomía) — para mapear/renombrar antes.
3. **Sin subcategoría** (ni subcat ni categoría) — quedan como están / decidís aparte.

## Test / corrida (usuario)
1. Dry-run en **emulador** con seed: revisar los tres grupos; que "A completar" tenga sentido (subcat → su padre real).
2. `--apply` en emulador → en "Por categoría" del Dashboard esos gastos ya aparecen bajo su categoría.
3. Repetir en **prod** (dry-run → `--apply`).
