# F9.30 — Cloud Function agendada: refresco diario del dólar MEP

> Reglas: doc-first · una PR. **Toca Functions** (no UI). Replica el trigger legacy de TC
> en Firebase. El usuario despliega/agenda; la función corre sola después. Ver
> `docs/tc-mep-fuente.md`.

## Objetivo
Mantener `tcDiario` actualizado automáticamente con el dólar MEP, como hacía el legacy
(`40_TC_Dolar.gs`, trigger 09:00). Hoy el rebuild solo **lee** `tcDiario` (`tcParaFecha`) y
lo llena por seed → sin refresco, el TC queda estático. Esta función cierra ese gap.

## Fuente
- API pública **`https://dolarapi.com/v1/dolares/bolsa`** (dólar MEP/Bolsa).
- Tomar el campo **`venta`** como `tcUsdArs` (mismo criterio que el legacy
  `gf_fetchMEP_Bolsa_`). Validar que sea número finito > 0.

## Tarea
1. **`onSchedule`** (firebase-functions v2/scheduler), región **`southamerica-east1`**
   (igual que el resto), todos los días ~09:00 `America/Argentina/Buenos_Aires`.
2. **Fetch** a dolarapi bolsa; parsear `venta`.
3. **Upsert** en `tcDiario`: doc id = **`YYYY-MM-DD`** del día (zona AR), campos
   `{ tcUsdArs: venta, actualizadoEn: serverTimestamp(), origen: 'dolarapi-bolsa' }`.
   Mismo shape que el seed (`scripts/seed/transformers/tcDaily.ts`) y que lee
   `tcParaFecha` (`src/datos/tcDiario.ts`).
4. **Tolerancia a fallo**: si la API no responde o el valor es inválido, **no romper** ni
   sobrescribir con basura — loguear y dejar el último válido (la función puede reintentar
   al día siguiente). No hace falta retry intra-día, pero registrar el error.
5. **Idempotente**: correrla dos veces el mismo día deja el mismo doc (upsert por fecha, no
   append).
6. Sin secrets (la API es pública). No tocar la lógica de lectura existente.

## Aceptación
- Existe una función agendada diaria en `southamerica-east1` que escribe el TC del día en
  `tcDiario` desde dolarapi bolsa.
- El doc tiene id `YYYY-MM-DD` (zona AR) y `tcUsdArs` numérico; `tcParaFecha` lo encuentra
  sin cambios.
- Falla de API → log + sin escritura corrupta; el último TC válido sigue disponible.
- Re-ejecución el mismo día = idempotente.

## Notas / decisiones
- **Tipo de dólar**: el legacy usa `bolsa`/`venta` (MEP). Confirmar que sigue siendo el
  correcto antes de desplegar; si se quisiera otro (oficial, blue), es cambiar el endpoint.
- **Respaldo manual**: la pantalla "Tipo de cambio" (F9.28) escribe el mismo
  `tcDiario` para forzar/corregir un día si la API falla. Esta función y esa pantalla
  comparten almacén — no duplicar fuentes.
- **Despliegue**: lo corre/agenda el usuario (deploy de Functions). Verificar una corrida
  manual antes de confiar en el schedule.
