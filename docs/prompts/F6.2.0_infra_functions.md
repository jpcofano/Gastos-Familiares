## Objetivo

Parar el paquete `functions/` y una function disparada por Firestore que:
1. se dispara al crearse un `comprobantes/{hash}`,
2. baja el archivo de Storage por `refStoragePdf`,
3. marca el comprobante como `'extraido'` con un **stub** (datos placeholder).

**Esta fase NO llama a Anthropic.** Prueba el pipeline punta a punta (trigger → download → writeback) sin dependencia externa. La extracción real es F6.2.1.

## Decisiones ya tomadas (no reabrir)

- **Trigger**: Firestore `onDocumentCreated('comprobantes/{hash}')`, **Gen 2**, region `southamerica-east1`. (Descartado el trigger de Storage `onFinalize`: ata al path del archivo y parte el dedup en dos lugares.)
- **Idempotencia**: doc-id = hash → re-subir no recrea el doc → no re-extrae. El writeback es un `update`, no re-dispara `onCreate`.
- **Guard**: la function corre **solo si `estado === 'subido'`**. Cualquier otro estado → `return` temprano.
- **Sin retry automático** por ahora. Un fallo deja el doc en `'error'` y se reintenta a mano (o F6.2.x).
- **Modelo de error explícito**: agregar `'error'` al enum + `errorExtraccion?: string`. Un fallo NO debe dejar el comprobante mintiendo en `'subido'`.

### Function `extraerComprobante`
```
onDocumentCreated('comprobantes/{hash}', region southamerica-east1):
  - snap = event.data; if (!snap) return
  - comp = snap.data()
  - GUARD: if (comp.estado !== 'subido') return
  - try:
      - bajar bytes del archivo de Storage por refStoragePdf (Admin SDK)
      - (stub) validar que se leyeron bytes (length > 0)
      - update: { estado: 'extraido', datosExtraidos: <placeholder>, errorExtraccion: FieldValue.delete() }
  - catch (e):
      - update: { estado: 'error', errorExtraccion: String(e?.message ?? e) }
```
- Options de la function: `timeoutSeconds: 120`, `memory: '512MiB'` (un archivo de 10MB en base64 pesa).
- El placeholder de `datosExtraidos` puede ser todo `null` + `moneda:'ARS'` + `numeroOperacion:'STUB'`; solo prueba el writeback.
