# F9.46 — Calendar: gate global `calendarSync` (corrige el toggle de F9.45)

> Reglas: doc-first · una PR chica. Corrige cómo se prende/apaga el Canal B de Calendar.
> NO reescribe la Function (F9.45 ya la tiene) — solo cambia el **gate** y agrega el switch.

## Problema
F9.45 quedó gateado **por ítem** (`itemEsperado.autoCalendario:true`). Pero el seed deja ese
campo siempre en `false` y no hay UI para tildarlo → hoy el Canal B **no sincroniza nada**.
La decisión de producto es un **toggle global del admin** (es un calendario compartido,
`juanymaypagos`): el admin lo prende y se crean los eventos de todos los vencimientos. El kit
ya tiene ese switch en `Perfil › Notificaciones` ("Eventos en Google Calendar").

## Tarea
1. **Flag global**: `config/familia.calendarSync: boolean` (default `false`).
2. **Callable admin-only** `setCalendarSync(boolean)` — mismo patrón auth→`/autorizados`→
   escribir que las configs F9.36–41. Es lo que mueve el switch del kit.
3. **Cambiar el gate de la Function** `sincronizarRecordatoriosCalendar`:
   - Al inicio, si `config/familia.calendarSync !== true` → no crear/actualizar nada y, si
     había eventos creados, **borrarlos** (por `calendarEventId` guardado en cada doc) y
     limpiar el campo. Salir.
   - Si `=== true` → sincronizar **todos** los vencimientos próximos (itemsEsperados +
     resúmenes de tarjeta no confirmados), sin depender de `autoCalendario`.
4. **`autoCalendario` por ítem**: dejarlo como **opt-out futuro** (no se usa para gatear
   ahora). No agregar UI para tildarlo en esta PR.
5. **UI**: cablear el switch de `Perfil › Notificaciones` (kit) a leer `calendarSync` y
   escribir vía `setCalendarSync` (solo admin; el dependiente ve estado Activo/Inactivo).

## Aceptación
- Admin mueve el switch en `Perfil › Notificaciones` → `config.familia.calendarSync` cambia.
- Con `calendarSync:true` y la Function corrida (`sincronizarCalendarAhora`), aparecen los
  eventos de los vencimientos en el calendario de `juanymaypagos`.
- Con `calendarSync:false`, una corrida borra los eventos futuros que había creado.
- Sin tocar la lógica de creación/idempotencia de F9.45 (solo el gate + el switch).

## Nota
- Probar **primero** `sincronizarCalendarAhora` con `calendarSync:true` para confirmar que el
  OAuth/refresh token crea el evento, antes de confiar en el cron diario.
