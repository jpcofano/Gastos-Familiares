# F9.16 — Modernizar gráficos de categoría + microcopy + color de "Salidas"

> Reglas: doc-first · una PR por fase · sin tocar Firestore/Functions · móvil primero.

## 1. "Por categoría": donut → barra apilada + ranking (Mensual y Anual)
El gauge/semidonut + leyenda es un recurso de dashboard viejo y la leyenda cortaba a
pocas categorías. Reemplazar por:
- Una **barra apilada** horizontal full-width (segmentos por categoría, color de cada
  una) que da la distribución de un vistazo.
- Debajo, el **ranking** de categorías con barra de progreso + monto + count.
- **Mensual**: top 6 + una fila **"Otras"** que agrupa el resto (con su count y suma).
- **Anual**: total arriba a la derecha + barra apilada + ranking completo.

## 2. Color de "Salidas" (outflow) — nuevo token
El amarillo `#f5a623` no convence. Introducir un token **`--gf-out`** para el color de
"Salidas" en gráficos (distinto de `--gf-expense`, el rojo de gasto negativo).
- Default: **naranja `#f97316`**.
- Reemplazar TODOS los `#f5a623` semánticos de salidas (cards Ingresos/Salidas, línea
  de promedio diario, leyenda + barras de "Ingresos y salidas por mes") por `var(--gf-out)`.
- Dejar como tweakeable la dupla **ingreso/salida**. Paletas sugeridas a ofrecer:
  - Marca (actual): income `#0c8f62` · out `#f97316`
  - A: `#10b981` · `#f97316`
  - B: `#06b6d4` · `#ec4899`
  - C: `#22c55e` · `#ef4444`
  - Editorial (propuesta): income `#047857` · out `#d97706` (esmeralda profundo + ámbar,
    sobrio, alineado al rumbo ink/esmeralda 2026)

## 3. Microcopy
- "Faltan ítems por cargar. 1" → **"Falta 1 ítem por cargar."**
- KPIs de Resumen: "Pesos disponibles" → **"Disponible (ARS)"**; "Faltante (USD)" →
  **"Resultado (USD)"** (el valor puede ser positivo; "faltante" engañaba).
