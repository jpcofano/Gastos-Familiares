# F9.52 — Etapa C: Open Graph para el preview de WhatsApp (y links compartidos)

> Reglas: **solo `index.html` (tags `<meta>`) + subir 1 imagen a `public/`.** Cero JS, cero
> modelo, cero lógica. Hace que cuando alguien pegue el link de la app en WhatsApp/Telegram/
> Slack aparezca una **tarjeta con ícono + título** en vez del link pelado.

## Diagnóstico
- La imagen OG ya está diseñada: **`exports/share/og-image.png`** (1200×630, ícono "$"+barras
  sobre ink + título). Hay que copiarla al repo y declararla.
- `index.html` hoy no tiene tags `og:*` ni `twitter:*` → preview pobre al compartir el link.
- El dominio final es **`https://gastos-familiares-jmsf.web.app`** (usar URLs absolutas: los
  crawlers de WhatsApp/Twitter no resuelven rutas relativas).

## Tarea

### 1. Subir la imagen
Copiar `exports/share/og-image.png` (de este repo de diseño) → **`public/og-image.png`** en el
repo de la app.

### 2. `index.html` — agregar en `<head>`
```html
<!-- Open Graph -->
<meta property="og:type" content="website" />
<meta property="og:site_name" content="Gastos Familiares" />
<meta property="og:title" content="Gastos Familiares" />
<meta property="og:description" content="Control de gastos de la familia" />
<meta property="og:url" content="https://gastos-familiares-jmsf.web.app/" />
<meta property="og:image" content="https://gastos-familiares-jmsf.web.app/og-image.png" />
<meta property="og:image:width" content="1200" />
<meta property="og:image:height" content="630" />
<meta property="og:locale" content="es_AR" />
<!-- Twitter -->
<meta name="twitter:card" content="summary_large_image" />
<meta name="twitter:title" content="Gastos Familiares" />
<meta name="twitter:description" content="Control de gastos de la familia" />
<meta name="twitter:image" content="https://gastos-familiares-jmsf.web.app/og-image.png" />
```
- `og:url`/`og:image` **absolutas** al dominio final. Si más adelante hay dominio propio,
  actualizar ambas.

## Aceptación
- `public/og-image.png` existe; `npm run build && firebase deploy --only hosting` la publica
  en `https://gastos-familiares-jmsf.web.app/og-image.png` (abre directo en el navegador).
- Pegar el link en WhatsApp (o validar en una herramienta de OG preview) muestra **tarjeta con
  imagen + título + descripción**, no el link pelado.
- No se tocó nada fuera de `index.html` + `public/og-image.png`.

## Notas
- WhatsApp **cachea** el preview por URL: si se prueba antes de subir la imagen, después hay
  que forzar recache (cambiar query `?v=2` una vez, o usar un debugger de OG).
- Dejar registrado en `docs/RUNBOOK-migracion-prod.md`, bloque "PWA (Etapa C)", el paso de
  subir la imagen + verificar el preview.
