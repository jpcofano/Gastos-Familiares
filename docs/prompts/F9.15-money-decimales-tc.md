# F9.15 — Pulido: Money sin decimales en mobile + reconciliar TC

> Reglas: doc-first · una PR por fase · móvil primero.

## 1. `Money` sin decimales en mobile
El componente `Money` del design system mostraba 2 decimales (`$ 2.018.435,00`).
- Sumar prop `decimals?: number` (default **2**, no rompe nada existente).
- En **todas** las vistas mobile que muestran montos grandes (visor de tarjetas,
  config de categorías, esperados, KPIs), pasar `decimals={0}`.
- Dashboard/Resumen ya van sin decimales vía `fmtMoney`; alinear el resto.

## 2. Reconciliar el tipo de cambio (IMPORTANTE)
F9.8 quedó con `TC_DEFAULT = 1180`, que era el valor **inventado** del mock viejo.
El valor **real** (semilla Firestore, hoja Config / ResumenMes) es:
- **TC USD→ARS ≈ 1454** (último MEP). Config lo expone como "TC USD→ARS (último MEP)".

Acción:
- En el repo, el TC **no** debería estar hardcodeado en `money.ts`. Debe leerse de la
  config real (`tipoCambio.valor` de Firestore). El `TC_DEFAULT` es solo fallback;
  si se deja un literal, que sea **1454**, no 1180.
- Verificar que Dashboard, Resumen y cualquier conversión usen esa única fuente.

PR visual + pequeño ajuste de config (sin tocar Functions).
