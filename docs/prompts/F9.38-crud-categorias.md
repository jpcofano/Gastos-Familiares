## Tarea (`Perfil › Categorías`)
Taxonomía **Categoría → Subcategoría → Etiqueta** (3 niveles; ya se lee real).
- **Alta / editar / baja** en cada nivel.
- Renombrar propaga (o mantiene id estable y solo cambia el label — preferible: **id
  estable, label editable**, así los movimientos no se huérfanan).
- Reordenar (opcional) para el orden en los gráficos.

## Integridad (clave)
- Los `movimientos` referencian categoría/subcat/etiqueta. **No** borrar en duro un nodo con
  movimientos asociados: o se **desactiva**, o se ofrece **reasignar** los movimientos a otro
  nodo antes de borrar. Confirmar política (recomendado: desactivar + advertir conteo).
- El **clasificador** (functions de aprendizaje) usa la taxonomía — un rename por label no lo
  rompe si el id es estable; un borrado sí. Avisar si afecta.
- "Otras" / fallback: mantener el nodo neutro para lo no clasificado.

## Aceptación
- Admin edita los 3 niveles, persistido; ids estables (los movimientos no se huérfanan).
- Borrar un nodo con uso → bloqueado o con reasignación, nunca deja movimientos colgados.
- Dashboard/Resumen siguen agrupando bien tras editar. Build verde.
