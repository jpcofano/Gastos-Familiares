> `src/vistas/perfil/Notificaciones.tsx` (`useRecordatorios`, `recordatoriosEsperados/Tarjetas`).
> El render ya está a paridad (F9.80) y **la ventana de 14 días es correcta** — no tocarla. Esto es 100% datos:
> hay vencimientos dentro de los 14 días pero la derivación devuelve 0.

---

## Qué mirar (condiciones exactas que un recordatorio necesita)

Un ítem entra a `recordatorios` solo si pasa TODOS estos filtros. El vacío = algún campo no está como se espera.

**Esperados** (`recordatoriosEsperados`, solo admin):
1. `item.tipo === 'Gasto'` (ojo mayúscula/valor exacto del enum).
2. `item.diaVencimiento != null` (número 1–31).
3. El estado del checklist NO es `cubierto` / `programado` / `no_registrado` / `no_aplica`.
   → Si el ítem ya figura **pagado/cubierto** este mes, se excluye (correcto, pero puede explicar el vacío si el
   match esperado↔movimiento lo está marcando cubierto de más).
4. `fecha = new Date(año, mesActual, diaVencimiento)` cae dentro de `[hoy, hoy+14]`.

**Tarjetas** (`recordatoriosTarjetas`):
1. Resumen con `estado !== 'confirmado'` **y** `fechaVencimiento` seteada, dentro de 14 días; o
2. `config.tarjetas[].venceDia` seteado y sin resumen del período cargado.

## §1 · Log de diagnóstico (temporal) en `useRecordatorios`

```ts
console.debug('[notif]', {
  hoy: hoy.toISOString().slice(0,10),
  mesActual: mesActualStr(),
  esAdmin,
  items: items.length,
  itemsGastoConVenc: items.filter(i => i.tipo === 'Gasto' && i.diaVencimiento != null).length,
  itemsGastoConVencEjemplo: items.filter(i => i.tipo === 'Gasto' && i.diaVencimiento != null)
    .slice(0,5).map(i => ({ cat: i.categoria, dia: i.diaVencimiento })),
  movimientos: movimientos.length,
  resumenes: resumenes.length,
  resumenesPend: resumenes.filter(r => r.estado !== 'confirmado' && r.fechaVencimiento).length,
  tarjetasCfgConVence: (config?.tarjetas ?? []).filter(t => t.venceDia).length,
  recEsperados: recEsperados.length,
  recTarjetas: recTarjetas.length,
});
```

Leer la consola en el dispositivo/navegador y decidir según qué está en 0:

- **`itemsGastoConVenc` = 0** → los pagos esperados no tienen `diaVencimiento` (o `tipo` no es exactamente `'Gasto'`).
  Es el origen más probable. Verificar el shape real de `ExpectedItem` en Firestore vs el nombre del campo que lee
  el código (`diaVencimiento`). Si en los docs el campo se llama distinto (ej. `venceDia`, `diaVence`), **ahí está el bug**:
  alinear el lector. Si el campo existe pero está vacío, cargarlo en Perfil › Pagos esperados / seed.
- **`itemsGastoConVenc` > 0 pero `recEsperados` = 0** → los está filtrando el estado del checklist (cubierto) o la
  ventana. Loguear también el `estado` de cada checklist para esos ítems y confirmar que no se marquen cubiertos de más.
- **Tarjetas todo en 0** → cargar `venceDia` en `config.tarjetas` (F9.35/F9.41) o `fechaVencimiento` en los resúmenes.

## §2 · Fix
