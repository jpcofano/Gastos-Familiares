# F6.4.5 — Addendum 4 (resolvedor de alias de miembros)

> Modifica base + addendums 1/2/3. Leer los cinco. No reescribir. Resuelve nombres crudos a miembro.

## Motivo
Un mismo miembro aparece escrito de formas distintas: "FEDERICO NICOLAS COFANO LASCANO" (PDF de tarjeta),
"Federico" (transferencia), "Fede". Todas deben resolver al miembro canónico (id `fede`).

## Decisiones cerradas
- **Los alias viven en `config/familia`** (identidad de los miembros), NO en el diccionario (que es
  aprendizaje de gastos — son cosas distintas).
- Regla de uso en tarjeta: **`tarjetaCodigo` DECIDE, el alias VALIDA** (ver integración con F6.5 abajo).

## 1. Shape de alias en `config/familia`
Agregar `alias` por miembro (array de strings, en minúscula sin acentos):
```
miembros: [
  { id: 'fede', nombre: 'Federico', alias: ['fede','federico','federico nicolas','federico nicolas cofano lascano'] },
  { id: 'juan', nombre: 'Juan',     alias: ['juan','juan pablo','juan pablo cofano'] },
  { id: 'maria', nombre: 'María',   alias: ['maria','maria ...'] },
  { id: 'sofia', nombre: 'Sofía',   alias: ['sofia','sofia ...'] }
]
```
- El `nombre` canónico cuenta como alias implícito. Reportar en PASO 1 el shape real de `config/familia`
  para encajar `alias` sin romper lo existente (auth/member resolution de F3 usa este doc).

## 2. Resolvedor `resolverMiembro(nombreCrudo)`
- Normaliza el input (lowercase, sin acentos, colapsar espacios) y matchea contra los alias de cada
  miembro → devuelve `memberId` o `null` si no hay match.
- Match por igualdad de alias normalizado; si querés tolerancia, `includes` como fallback (un alias
  contenido en el nombre crudo), pero igualdad primero para evitar falsos positivos.
- Puro, sin SDK. Cargado desde `config/familia` (que ya se lee al iniciar sesión).

## 3. Integración (dónde se usa)
- **Comprobante suelto / manual**: persona = quién subió (memberId de sesión). NO usa el resolvedor
  (ya es un memberId). Sin cambios.
- **Resumen de tarjeta (F6.5 — solo anotar, no implementar acá)**:
  - Persona del consumo = **titular de la tarjeta vía `tarjetaCodigo`** (catálogo) → fuente que DECIDE.
  - El resolvedor se usa para **validar**: parsear el titular impreso en el PDF → `resolverMiembro()` →
    confirmar que coincide con el titular del `tarjetaCodigo`. Si NO coincide → advertir (puede ser
    adicional o detección errónea de tarjeta). Alias VALIDA, tarjetaCodigo DECIDE.

## Cierre
- Prueba: `resolverMiembro('FEDERICO NICOLAS COFANO LASCANO')`, `resolverMiembro('Fede')`,
  `resolverMiembro('federico')` → todos devuelven `fede`. Un nombre desconocido → null.
- Anotar en CLAUDE.md: alias en config/familia; regla "tarjetaCodigo decide, alias valida" para F6.5.
