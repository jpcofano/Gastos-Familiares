# F3.1 — Init del proyecto cliente (Vite + React + TS)

**Fase:** F3 (Auth + shell PWA) · **Sub-paso:** 3.1
**Scope:** solo emulador. Nada toca producción en este paso.

---

## Contexto

Monorepo de migración del sistema de gastos familiares a Firebase. El seed (F2) está cerrado y validado (8 OK / 0 FAIL contra emulador). Ahora arranca F3: cliente web.

Stack: Vite + React + TypeScript, mismo blueprint que el proyecto de comidas.

Decisiones cerradas relevantes para este paso:
- Todo vive en este mismo repo (monorepo).
- Trabajamos solo contra emulador por ahora.
- Firebase Auth + Google Sign-In, sin custom claims.
- F3.3 lee `/config/familia` con el SDK del cliente directo (sin Cloud Functions, para evitar costos).

---

## Antes de escribir código — investigá y reportá

1. **Estructura actual del repo:** dónde viven `transformers/`, `validators/`, `firestore.rules`, `firestore.indexes.json`, y si ya hay un `package.json` raíz o es por carpeta. Quiero saber dónde conviene meter el cliente sin pisar nada del seed.
2. **Tipos del modelo F1:** si los tipos que usan los transformers están en un lugar reusable o acoplados al seed. Necesito saber si el cliente puede importarlos o si hay que extraer un paquete de tipos compartido.
3. **Entorno:** versión de Node y si hay workspaces configurados (npm / pnpm / yarn).

**Esperá mi aprobación del plan de estructura antes de tocar archivos.**

---

## Cuando apruebe el plan, F3.1 hace

- Inicializa Vite + React + TS en una subcarpeta del monorepo (la que acordemos tras el reporte).
- Instala Firebase SDK del cliente.
- Crea la inicialización de Firebase (`app`, `auth`, `firestore`) con conexión condicional al emulador:
  - Si `import.meta.env.DEV`: `connectAuthEmulator` y `connectFirestoreEmulator` con los puertos del `firebase.json` actual.
  - En prod: config real.
- Un componente raíz mínimo que solo confirme "conectado al emulador" en consola.
- Scripts npm para levantar el dev server.

---

## Restricciones

- **No** toques `transformers/`, `validators/`, ni las Rules en este paso.
- **No** corras `firebase deploy` ni nada contra producción.
- **No** corras `clasp` (esto ya es Firebase, no Apps Script).

---

## Criterio de cierre

`npm run dev` (o el script que definas) levanta, la consola muestra conexión al emulador de Auth y Firestore, y no hay imports rotos.
