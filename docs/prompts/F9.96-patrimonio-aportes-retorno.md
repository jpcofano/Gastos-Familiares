# F9.96 — Patrimonio: aportes/retiros + retorno real aproximado (TWR)

## Contexto

La Evolución hoy muestra cambio de valor, no retorno: un aporte parece
ganancia. Este prompt agrega el registro de flujos y el cálculo de retorno
por período. **Razón para construirlo YA aunque haya pocos snapshots:** los
flujos hay que registrarlos cuando ocurren; si la feature llega tarde, los
flujos viejos se pierden y el retorno histórico queda contaminado para
siempre.

## 1) Colección `flujosPatrimonio/{autoId}`

`{ fecha: Timestamp, tipo: 'aporte' | 'retiro', montoUsd: number,
   cuenta: string | null, nota: string }`

Reglas: `esDueno()`. Un retiro se guarda con monto positivo y `tipo` lo
distingue (no montos negativos).

## 2) UI — card "Aportes y retiros" (solapa Configuración)

Lista desc por fecha (fecha · tipo · monto · cuenta · nota) + alta/edición/
borrado con confirmación. Simple, sin categorías.

## 3) Cálculo de retorno por período

Entre cada par de snapshots consecutivos (V0 → V1) con flujo neto F del
período (aportes − retiros entre ambas fechas):

- **Modified Dietz con peso 0,5** (asumir flujos a mitad de período):
  `R = (V1 − V0 − F) / (V0 + 0,5·F)`
- Encadenado para el acumulado: `(1+R1)·(1+R2)·…−1`.
- Si F = 0 en el período, queda el retorno simple de siempre.

Implementar como función pura testeable en `analysis`/util, no en el
componente.

**Honestidad metodológica (mostrar):** es aproximación — con corridas
quincenales/mensuales y flujos a mitad de período asumidos, el error es
aceptable pero existe. Etiquetar el número como "Retorno (aprox.)".

## 4) Dónde se muestra

- Card Evolución (Resumen): a la línea de Δ valor se agrega
  "Retorno (aprox.): X%" por período y el acumulado de la serie.
  La nota actual "no descuenta aportes ni retiros" se reemplaza por
  "descuenta los flujos registrados en Configuración (aprox. Dietz)".
  Si NO hay flujos registrados en toda la serie, conservar la nota vieja.
- Informe PDF, sección Evolución: columna "Retorno aprox." cuando existan
  flujos.

## Test

1. Sin flujos: todo se ve como hoy (nota vieja incluida).
2. Registrar un aporte de 1.000 entre dos snapshots: el Δ valor no cambia,
   el Retorno (aprox.) del período baja respecto del Δ%, y el acumulado se
   recalcula. Verificar contra cálculo a mano de la fórmula.
3. Un retiro invierte el efecto.
4. Borrar el flujo restaura los números originales.
