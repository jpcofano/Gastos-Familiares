# F8.4 — Cleanup: quitar placeholders redundantes (Subcategorías / Etiquetas)

## Contexto / decisión

En F8.1 el grupo "Clasificación y aprendizaje · admin" quedó con 5 ítems, dos de ellos
con badge PRONTO: **Subcategorías** y **Etiquetas**. Al auditar el repo se confirmó que
**esas dos colecciones ya tienen CRUD admin completo dentro de `Perfil › Categorías`**
(`src/vistas/perfil/Categorias.tsx`, F9.38): subcategorías anidadas bajo cada categoría y
etiquetas en su propia sección, con cascade server-side de renombrado a `movimientos` y
`diccionario` (callable `guardarTaxonomia`).

→ **No se construye ningún editor nuevo.** Se eliminan los dos placeholders (serían
duplicados) y se mejora la descubribilidad del ítem Categorías. Con esto el grupo del
motor queda **completo**: Diccionario (F8.1), Destinos (F8.2), Normalización (F8.3).

Frontend puro, sin backend, sin rules. Deploy: `--only hosting`.

> Este prompt asume F8.1–F8.3 ya aplicados. Los ítems Destinos y Normalización ya tienen
> `onClick` (no PRONTO). Solo quedan con PRONTO los dos que este prompt borra.

---

## Cambio 1 — `src/vistas/Perfil.tsx`: borrar los 2 ítems y fijar `last`

Tras F8.3, el grupo termina así (Subcategorías y Etiquetas siguen con PRONTO):

BUSCAR:
```tsx
    <Item icon="sparkles" title="Normalización" desc="Limpieza previa al matching (prefijos, regex)" onClick={() => navigate('/perfil/normalizacion')} />
    <Item icon="layers" title="Subcategorías" desc="Árbol de subcategorías por categoría" right={<Pronto />} />
    <Item icon="tag" title="Etiquetas" desc="Etiquetas libres para movimientos" right={<Pronto />} last />
```
REEMPLAZAR:
```tsx
    <Item icon="sparkles" title="Normalización" desc="Limpieza previa al matching (prefijos, regex)" onClick={() => navigate('/perfil/normalizacion')} last />
```

> Normalización pasa a ser el último ítem del grupo → toma `last` (sin borde inferior).

## Cambio 3 — `src/vistas/Perfil.tsx`: descubribilidad del ítem Categorías (opcional pero recomendado)

Como subcategorías y etiquetas viven dentro de Categorías, señalarlo en el subtítulo para
que no queden "escondidas". El `desc` hoy es dinámico (`descCategorias` = "N categorías").

BUSCAR:
```tsx
  const descCategorias = `${config?.categorias.filter(c => c.activo).length ?? 0} categorías`;
```
REEMPLAZAR:
```tsx
  const descCategorias = `${config?.categorias.filter(c => c.activo).length ?? 0} categorías · subcategorías y etiquetas`;
```
