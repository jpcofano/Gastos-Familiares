# F9.11 — Subcategorías en la UI (taxonomía de 3 niveles)

> Reglas: doc-first · una PR por fase · sin tocar Firestore/Functions · móvil primero.
> Contexto de datos: `docs/modelo-datos-real.md` (Categoría → Subcategoría → Etiqueta).

El modelo real es de **3 niveles**: `Categoría → Subcategoría → Etiqueta`. Las
subcategorías ya existen en los datos (p. ej. Casa → Expensas/Luz/Gas/Internet/Agua/ABL;
Educación y chicos → Colegio Fede/Colegio Sofi/Actividades extracurriculares).

## UI
En **Perfil › Categorías**, cada categoría es una fila **expandible**:
- La fila muestra: color, nombre, "N movimientos · M subcat.", gasto del mes, y un
  chevron que rota (down) si tiene subcategorías (right si no).
- Al tocar, despliega las subcategorías como **chips** (pills gris claro, `nowrap`),
  indentadas bajo la categoría.
- Solo una abierta a la vez (acordeón).

## Dónde más usar el 3er nivel
- En el detalle de un movimiento / gasto esperado, mostrar `Categoría · Subcategoría`
  y la **Etiqueta** (tag transversal: hijos, rutina-casa, rutina-trabajo) como chip aparte.
- El alta/edición de movimiento: selector de subcategoría dependiente de la categoría.

PR visual (la estructura de datos ya existe en Firestore: campos Categoría/Subcategoria/
Etiqueta en Gastos_Manuales y GastosEsperados).
