## §1 · FIX BLOQUEANTE — regla de Storage para informes

`generarYArchivarInforme()` sube a `patrimonio/informes/{...}.pdf`, pero
`storage.rules` no tiene match para ese path → el upload falla con
`permission-denied` (la descarga local funciona, el archivado no).

Agregar a `storage.rules`:

```
match /patrimonio/informes/{archivo} {
  // Control fino por rol cae en Firestore (informesPortafolio, esDueno).
  // Mismo criterio documentado en /comprobantes.
  allow read:   if request.auth != null;
  allow write:  if request.auth != null
                && request.resource.size < 10 * 1024 * 1024
                && request.resource.contentType.matches('application/pdf');
  allow delete: if false;
}
```

Recordar en el resumen final: requiere `firebase deploy --only storage`.
