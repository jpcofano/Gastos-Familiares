# F9.82.1 — Hotfix pase débil: índice `destinos` + fail-soft

## Contexto
En prod, `matchComprobante` crashea con `FAILED_PRECONDITION: The query requires an index` al ejecutar `cargarNombresDestinoAprendidos()` (F9.82). La query `destinos where tipo == 'nombre' && confianza >= 0.7` requiere índice compuesto inexistente. Peor: el crash aborta TODO el match (ni rama 2/3 corre) — el comprobante queda colgado en "Calculando match…".

Dos fixes, ambos obligatorios.

## Cambio 1 — `firestore.indexes.json`: índice compuesto versionado

Agregar al array `indexes`:

```json
{
  "collectionGroup": "destinos",
  "queryScope": "COLLECTION",
  "fields": [
    { "fieldPath": "tipo",      "order": "ASCENDING" },
    { "fieldPath": "confianza", "order": "ASCENDING" }
  ]
}
```

## Cambio 2 — `functions/src/index.ts`: fail-soft en el pase débil

El pase débil es mejora opcional; su falla NO puede matar el match. En el bloque F9.82 (post pase fuerte con 0 candidatos):

BUSCAR:
```ts
      const nombresAprendidos = await cargarNombresDestinoAprendidos();
```
REEMPLAZAR:
```ts
      // F9.82.1 — fail-soft: si la query de alias falla (índice, permisos, etc.),
      // degradar a pase débil sin alias en vez de abortar todo el match
      let nombresAprendidos: Map<string, string> | null = null;
      try {
        nombresAprendidos = await cargarNombresDestinoAprendidos();
      } catch (e) {
        console.error('[matchComprobante] cargarNombresDestinoAprendidos falló, sigo sin alias:', e);
      }
```

(Si el nombre de variable/estructura driftó, adaptar preservando: try/catch alrededor de la carga de alias, `null` como fallback, el match continúa siempre.)
