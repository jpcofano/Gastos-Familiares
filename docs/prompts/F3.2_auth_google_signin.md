# F3.2 — Capa de auth: Google Sign-In

**Fase:** F3 (Auth + shell PWA) · **Sub-paso:** 3.2
**Prerequisito:** F3.1.1 y F3.1.3 cerrados (cliente inicializado, tipos corregidos, 8 OK / 0 FAIL).
**Scope:** solo emulador. Nada toca producción.

---

## Contexto

El cliente Vite + React + TS ya levanta y conecta a los emuladores (Auth 9099, Firestore 8080) vía `src/firebase.ts`. Ahora se agrega la capa de autenticación.

Decisiones cerradas que aplican:
- Firebase Auth con `signInWithPopup` + `GoogleAuthProvider`. Sin custom claims.
- Persistencia local del login (`browserLocalPersistence` — es el default del SDK web, solo confirmar que no se pisó).
- La whitelist y la resolución de `memberId` contra `/config/familia` son **F3.3, NO este paso**. En F3.2 cualquier cuenta Google puede loguearse; la autorización viene después.

---

## Qué hace F3.2

1. **`src/auth.ts`** (o ubicación equivalente coherente con la estructura existente):
   - `signInConGoogle()`: `signInWithPopup(auth, new GoogleAuthProvider())`. Devuelve el user o lanza el error (no tragarlo).
   - `signOutUsuario()`: `signOut(auth)`.
   - Manejo del caso popup cerrado por el usuario (`auth/popup-closed-by-user`): no es un error a loguear como falla, es cancelación.

2. **Hook `useFirebaseUser()`** (en `src/hooks/` o equivalente):
   - Se suscribe con `onAuthStateChanged(auth, ...)`.
   - Expone `{ user: User | null, cargando: boolean }`. `cargando` es `true` hasta el primer callback de `onAuthStateChanged` (evita el flash de "no logueado" al refrescar).
   - Cleanup correcto del listener en el unmount.

3. **UI mínima de prueba en `App.tsx`** (provisoria, F3.4 la reemplaza con el AppShell):
   - Si `cargando`: texto "Cargando…".
   - Si no hay user: botón "Ingresar con Google".
   - Si hay user: email del user + botón "Salir".

4. **Verificación del emulador de Auth:** el emulador de Auth permite crear cuentas falsas desde su UI al hacer el popup — el flujo completo se prueba sin tocar cuentas reales de Google. Documentar en el reporte cómo probarlo.

---

## Restricciones

- **NO** leer `/config/familia` ni resolver `memberId` (es F3.3).
- **NO** implementar los 4 estados del AppShell (es F3.4).
- **NO** tocar `scripts/seed/`, Rules, ni índices.
- **NO** tocar producción. El popup contra el emulador usa cuentas fake del emulador.
- Tipos: usar `User` del SDK de Firebase Auth directamente para el user crudo — NO crear un tipo de dominio para esto todavía (el tipo de dominio del miembro llega en F3.3 con `FamiliaMiembro`).

---

## Criterio de cierre

- Login con cuenta fake del emulador funciona: el email aparece en pantalla.
- Refresh de la página: sigue logueado (persistencia), sin flash de "no logueado".
- Logout funciona y vuelve al botón de ingreso.
- Cancelar el popup no rompe nada ni ensucia la consola con errores no manejados.
- `npm run validate` sigue en 8 OK / 0 FAIL (nada del seed se tocó).
