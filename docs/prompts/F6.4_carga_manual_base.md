# F6.4 — Carga manual de movimiento (base)

> Modo B: este es el base congelado. Cambios posteriores → addendum numerado, no reescribir.
> Emulador-only. Cierre toca docs/CLAUDE.md en el mismo commit.

## Motivo
Gastos en efectivo o sin comprobante (propina, algo que no sabés qué compraste), cargados a mano
desde la página Comprobantes. Variante 1: **alta suelta, sin comprobante, ingreso o gasto.**

## Decisiones cerradas
- Reusa el componente `AltaMovimiento` existente, en **modo vacío** (sin preload, sin `hashPdf`,
  sin `itemEsperadoId`).
- **Número de operación determinístico** estilo extracción `YYYY-MM-<slug>`, por consistencia con los
  movimientos que vienen de comprobantes. El `<slug>` sale de la descripción/categoría (en efectivo no
  hay comercio). El formato hoy vive en el PROMPT de extracción (lo genera el modelo), no en código →
  para el alta manual se hace un **helper chico en cliente que replica ese mismo formato**. Nota en
  CLAUDE.md de que el formato vive en dos lugares (prompt + helper) y se mantienen consistentes a mano.
- **Dedup advisory, NO bloqueante**: certeza bloquea (mismo archivo, hashPdf), heurística avisa. Si el
  número generado coincide con uno ya existente, **mostrar aviso y dejar continuar**. NUNCA dropear
  silencioso.
- **Invariante**: el doc-id del movimiento sigue siendo único (auto-id). El número determinístico es un
  **campo**, no la clave → dos gastos legítimos con el mismo `YYYY-MM-cafe` coexisten.
- Marcar **origen** del movimiento como `'manual'` para distinguir de los extraídos.

## PASO 2 — Implementación (con aprobación)
- Botón **"Cargar movimiento manual"** en `/comprobantes` → abre `AltaMovimiento` en modo vacío,
  con selector ingreso/gasto.
- Helper cliente `generarNumeroManual(fecha, descripcion|categoria)` → `YYYY-MM-<slug>` (mismo formato
  que el pseudo-número de extracción).
- **Pre-write**: buscar si ya existe movimiento con ese número → si sí, aviso ("Ya hay un movimiento con
  este número, ¿cargar igual?") con continuar/cancelar. Continuar = se escribe igual.
- doc-id auto-id (único siempre). Número como campo. `origen: 'manual'`.
- Campos de comprobante (`hashPdf`, `itemEsperadoId`, `confirmadoPago`) no aplican → null/ausentes.
- El `onSnapshot` de F5.3 refleja el alta sola; no agregar refetch.
