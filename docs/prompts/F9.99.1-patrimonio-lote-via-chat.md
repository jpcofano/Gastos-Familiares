# F9.99.1 — Análisis de toda la cartera vía chat (modo lote)

> Prompt de feature para Code. Va a `docs/prompts/`.
> Cierra el hueco de F9.99: el camino chat cubre posición/sectorial/agenda pero
> "Analizar toda la cartera" quedó API-only (loop cliente de 34 llamadas,
> ~USD 1,7-2,7 por corrida). Este modo lo hace con costo cero.

## Alcance

### 1. Backend — `generarPromptIA` modo `lote`
- Aceptar `modo: 'lote'` con `contexto: { posiciones: [{ticker, sector, pesoEnCartera, valorUsd}], global: {...} }`.
- El prompt generado reutiliza `buildPromptPosicion` conceptualmente pero para N
  posiciones: mismas reglas innegociables, mismo shape JSON por ticker, con el
  contexto global de la cartera (concentraciones, drivers compartidos) una sola
  vez al inicio.
- Formato de salida pedido al chat: **un único bloque ```json** con
  `{ "analisis": [ { "ticker": "...", "resultado": { ...shape posicion... } } ] }`
  — un solo bloque para que `extraerResultado` lo capture entero.
- Instrucción de agrupar búsquedas web por driver (energía AR sirve para
  TRAN/PAMP/YPFD/VIST; soberano para los GD; cripto para BTC/ETH/AAVE/UNI).
- Instrucción de continuación: si no entra en un mensaje, cortar al final de un
  elemento completo del array y esperar "seguí" (el usuario concatena antes de
  importar, o importa por tandas).

### 2. Backend — `importarAnalisisIA` modo `lote`
- Aceptar `modo: 'lote'`; `extraerResultado('lote', ...)` parsea JSON igual que
  posicion/agenda.
- Validación: `analisis` array no vacío; cada elemento con `ticker` string y
  `resultado` que pase el validador de `posicion` existente
  (`queEs/situacionActual/riesgos/rolEnCartera`). Fail-soft POR ELEMENTO:
  importar los válidos, reportar los inválidos por ticker en el resumen
  (`"28 importados, 2 con error: XXX (campos faltantes...), YYY"`).
- Persistencia: batch-write a `analisisPosiciones/{ticker}` con
  `origen: 'chat'`, `modeloUsado: 'chat-manual'`, `generadoEn` server +
  `generadoEnISO` — idéntico al modo posicion.

### 3. UI
- Botón "Chat" en la card "Analizar toda la cartera" (mismo patrón visual que
  sectorial/agenda), disponible con toggle IA OFF.
- Reutilizar `ModalPromptChat` con `modo: 'lote'`: el contexto se arma igual que
  en `handleAnalizarLote` (mismo byTickerMap) más el contexto global.
- Al importar OK, refrescar `analisisCache` para todos los tickers importados;
  chip "vía chat" en cada análisis (ya existe).
- El texto "consume API · activar IA en Configuración" pasa a aplicar solo al
  botón "Analizar lote"; el botón Chat no lo requiere.

## Fuera de alcance
- No tocar el camino API del lote.
- No paginación server-side del prompt: la continuación es responsabilidad del
  chat/usuario.

## Criterios de aceptación
1. Con toggle IA OFF: generar prompt lote → pegarlo en un chat → pegar la
   respuesta completa → importa todas las posiciones válidas en una sola acción.
2. Un elemento inválido no aborta el lote; se informa por ticker.
3. Render de cada análisis idéntico al de origen api + chip "vía chat".
4. Los 4 modos (`posicion`/`sectorial`/`agenda`/`lote`) conviven sin regresión.
5. Prompt de esta feature a `docs/prompts/` y push al cerrar.
