# F9.8 — Currency engine: formateo único + un solo tipo de cambio

> Reglas: doc-first · una PR por fase · sin tocar Firestore/Functions · paridad
> funcional con la legacy · rumbo visual moderno 2026 (ink/esmeralda) · móvil primero.

Hoy cada pantalla formatea la moneda distinto (herencia de los reportes `.gs`
separados) y el Dashboard usa un tipo de cambio propio. Unificar:

## 1. Un solo tipo de cambio
- Toda conversión USD↔ARS usa **una** fuente: el TC de configuración
  (`M_TC` / `tipoCambio.valor`, hoy 1180).
- **Eliminar cualquier TC divergente** (el Dashboard tenía 1485 hardcodeado). Ningún
  componente define su propio TC.

## 2. Notación única (es-AR)
- ARS → `$ 1.234.567` · USD → `U$S 1.234`. Igual que el componente `Money` del design
  system y la pantalla Resumen.
- El Dashboard NO debe usar `USD 1.526`; usar `U$S`.
- Miles con punto; **sin decimales** en mobile (montos grandes).

## 3. Sin sufijo "eq" ni "≈"
- El monto secundario (la otra moneda) se muestra **solo el número formateado**, sin
  `eq` ni `≈`. Ej: principal `U$S 1.526` → secundario `$ 1.800.680` (no `$ 1.800.680 eq`).

## Implementación sugerida
Un helper único (`fmtMoney`) consumido por todas las vistas:
- `fmtMoney(monto, { from: 'USD'|'ARS', to: 'USD'|'ARS' })` → string formateado.
- El pill ARS/USD global elige `to`; el principal y el secundario salen del mismo helper.
- Reemplazar los formateadores locales de cada pantalla (Dashboard `curBig/curEq`,
  Resumen `fmtArs`, etc.) por este helper.

PR visual (sin tocar datos ni Functions).
