# F6.4.5 — Addendum 3 (mejoras sobre la paridad)

> Modifica base + addendum 1 + addendum 2. Leer los cuatro. No reescribir. Tres mejoras aprobadas.

## a) Umbral de confianza para el prellenado
- El diccionario tiene `confianza` (0-1). En la propuesta/alta:
  - `confianza ≥ 0.7` → **prellenar** cat/subcat/descripcionLimpia.
  - `confianza < 0.7` → dejar **en blanco, marcado para revisar** (no prellenar algo dudoso que el usuario
    no revise por parecer resuelto). El usuario completa a mano.
- Umbral 0.7 configurable en un solo lugar (constante), por si hay que ajustarlo.

## b) Persona sugerida — jerarquía de fuentes (CAMBIO sobre la idea original)
Persona NO sale del `personaDefault` del diccionario como fuente principal. Jerarquía:
1. **Quién subió el comprobante** (`subidoPor` / memberId de la sesión) → fuente PRINCIPAL. Si Federico
   sube/comparte, el gasto se sugiere como de Federico.
2. **`personaDefault` del diccionario** → solo fallback si no hay quién-subió (raro) o sugerencia
   secundaria.
3. **Siempre editable**: a veces se sube un comprobante de un gasto de otro (Juan sube algo de María).
- `monedaDefault` del diccionario sí entra como sugerencia directa (no tiene la ambigüedad de persona).

### Nota de alcance para F6.5 (NO implementar acá, dejar anotado)
En el resumen de tarjeta los consumos no tienen "quién los subió" individual → persona sale del
**titular de la tarjeta**, no de quién subió el PDF. Eje distinto. F6.5 NO debe copiar la regla
"persona = quién subió"; usa el titular. Anotarlo en CLAUDE.md para que F6.5 no se equivoque.

## c) La corrección del usuario pesa más
- Cuando el usuario **acepta** la sugerencia tal cual → `usoCount += 1` (como el viejo).
- Cuando el usuario **corrige** la sugerencia (cat/subcat/descripción distintas a lo sugerido) → además de
  actualizar el mapeo, **subir la `confianza`** de la entrada corregida (acercándola a 1, con tope), para
  que gane en lookups futuros. La corrección es la señal más valiosa; el viejo solo contaba uso.
- Si no existía mapeo para ese patrón normalizado, la corrección **crea** la entrada con confianza alta.

## Cierre (suma a addendums 1 y 2)
- Prueba umbral: una entrada con confianza baja → la propuesta queda en blanco para revisar; una con alta →
  prellena.
- Prueba persona: Federico sube un comprobante → persona sugerida = Federico (no lo que diga el texto);
  editable.
- Prueba corrección: corregir una sugerencia → verificar que la confianza de esa entrada subió y que el
  próximo lookup del mismo texto la prefiere.
