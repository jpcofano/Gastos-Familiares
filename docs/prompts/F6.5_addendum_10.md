# F6.5 — Addendum 10 (ajuste manual de cuadre, backstop de error de parser)

> Cuando el parser lee mal un monto o pierde una línea que NO se puede reconstruir del PDF, el cuadre
> bloquea la confirmación y el resumen queda trabado. Hace falta una válvula para cerrar la diferencia
> y seguir, sin saltear el cuadre ni tapar errores a ciegas. Fix: el residuo entra como una entrada
> más de `ajustesConsolidado` (mismo mecanismo que el DEV PER), marcada como manual para auditoría.

## Contexto (verificado en el repo)
- `AjusteConsolidado` (`src/types/index.ts:42`) = `{ concepto: string; montoARS: number; montoUSD: number }`.
  Ya tiene las dos monedas → un ajuste manual entra sin cambiar la forma.
- `calcularCuadre` (`src/datos/resumenesTarjeta.ts:167-194`) ya suma los ajustes al neto
  (`sumaARS += a.montoARS`, líneas 183-186) y compara contra el total PDF con tolerancia de $1.
- El DEV PER ya usa este mecanismo: monto que entra al cuadre, fuera de movimientos, visible en banner.
  El ajuste por error de parseo es la misma figura.

## Decisión (A-over-B)
- **A (elegida):** el residuo se modela como una entrada de `ajustesConsolidado` (un solo campo,
  reusado). Va fuera de movimientos, consistente con el DEV PER. No se crea campo nuevo. El cuadre NO
  se saltea: el residuo se hace explícito *dentro* del cuadre.
- **B (rechazada):** campo separado / saltear el gate del cuadre. B duplica mecanismo y rompe la regla
  del addendum 9 ("el cuadre es la única compuerta de integridad").
- El texto del `concepto` no es significativo; la distinción DEV PER vs manual se hace por un campo
  **estructurado** (`origen`), no por la etiqueta.

## Cambios

### 1. Discriminador estructurado en el tipo
`src/types/index.ts:42`:
```diff
 export interface AjusteConsolidado {
   concepto: string;
   montoARS: number;
   montoUSD: number;
+  origen?: 'pdf' | 'manual';   // 'pdf' = DEV PER extraído; 'manual' = ajuste de cuadre. undefined ⇒ 'pdf' (compat)
 }
```
Backward-compatible: los DEV PER existentes (sin `origen`) se tratan como `'pdf'`. El CF puede dejarlo
en undefined o setear `'pdf'` explícito; no es obligatorio tocarlo.

### 2. Función para agregar el ajuste manual
`src/datos/resumenesTarjeta.ts`, nueva función (espejo de `asignarTarjetaResumen`):
```ts
export async function agregarAjusteCuadreManual(
  resumen: CardStatement,
  memberId: string,
): Promise<Resultado<void>> {
  // residuo = total − suma actual, por moneda (la suma ya incluye DEV PER y demás ajustes)
  const cuadre = calcularCuadre(resumen.movimientosParseados, resumen.totalARS, resumen.totalUSD, resumen.ajustesConsolidado);
  const residuoARS = +(resumen.totalARS - cuadre.sumaARS).toFixed(2);
  const residuoUSD = +(resumen.totalUSD - cuadre.sumaUSD).toFixed(2);
  if (Math.abs(residuoARS) <= 1 && Math.abs(residuoUSD) <= 1) {
    return { ok: false, error: new Error('No hay diferencia para ajustar') };
  }
  const entrada: AjusteConsolidado = {
    concepto: 'Diferencia no identificada (ajuste manual)',
    montoARS: residuoARS,
    montoUSD: residuoUSD,
    origen:   'manual',
  };
  await updateDoc(doc(db, 'resumenesTarjeta', resumen.id), {
    ajustesConsolidado: [...resumen.ajustesConsolidado, entrada],
    actualizadoEn:      serverTimestamp(),
  });
  return { ok: true, data: undefined };
}
```
Tras el `updateDoc`, el `onSnapshot` re-renderiza el preview, `calcularCuadre` da verde y se habilita
confirmar. No se toca `calcularCuadre` ni el CF.

### 3. UI: botón + warning de diferencia grande
`src/vistas/ResumenesTarjeta.tsx`: cuando el cuadre NO cierra, mostrar botón "Cerrar diferencia
manualmente" que llama a `agregarAjusteCuadreManual`. **Antes de aplicar**, si el residuo supera un
umbral (sugerido: `> $5000 ARS` o `> 2%` del total, lo que sea menor; USD `> 2`), pedir confirmación:
"Diferencia grande — puede ser una línea real no leída, no un error de redondeo. ¿Absorber igual?".
El ajuste manual es el backstop para montos mal leídos, no el camino principal.

## No tocar
- `calcularCuadre` ni el guard de `confirmarResumenTarjeta`: el gate sigue intacto, ahora con un término
  residual nombrado en vez de obligar a reconstruir línea por línea.
- El CF: el ajuste es 100% del lado cliente, post-parseo.

## Auditoría
Un resumen que necesitó fudge se detecta por una entrada con `origen === 'manual'` en
`ajustesConsolidado`. Sumar al validator un check que liste estos resúmenes (señal de calidad del
parser; no debería crecer con el tiempo si el parseo está sano).

## Cierre
- Va **después del addendum 9**: el 9 elimina la causa más común de descuadre (dedup borrando líneas
  reales). El 10 es el backstop para lo que quede (un monto realmente mal leído e irreconstruible).
- Prueba: forzar un descuadre (editar un monto en el preview) → botón aparece → al cerrar, el cuadre
  da verde, la entrada manual queda en el banner, y confirmar crea los movimientos sin el residuo
  (que vive fuera de movimientos).
