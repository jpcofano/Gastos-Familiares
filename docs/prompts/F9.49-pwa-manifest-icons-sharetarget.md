# F9.49 — Etapa C: PWA instalable (manifest + íconos + share_target)

> Reglas: doc-first. **Toca `public/` (manifest + íconos), `index.html` (links iOS) y nada
> más.** Hace la app **instalable** y **destapa el Web Share Target** que ya está codeado en
> `public/sw.js` pero hoy es código muerto (no hay manifest que lo declare). Cero cambios de
> modelo, cero lógica nueva. Ver `docs/etapa-c-pwa.md`.

## Diagnóstico (auditado contra el repo)
- `index.html` linkea `<link rel="manifest" href="/manifest.webmanifest">` pero **el archivo
  no existe en `public/`** → la app **no es instalable** y el SO **no puede ofrecer
  `share_target`**.
- `public/sw.js` ya maneja `POST /share-target` (guarda el File en IndexedDB y redirige a
  `/comprobantes?share=1`), y `src/datos/shareTargetIdb.ts` lo lee-y-borra. **Pero sin
  declarar `share_target` en el manifest, el share sheet del SO nunca apunta a la app** → ese
  handler nunca corre. Esta tarea lo conecta.
- Solo existe `public/icons/icon.svg` (un "GF" blanco sobre `#1e3a5f`). Falta el set PNG que
  pide la instalación (192/512 + maskable) y el `apple-touch-icon`.

## Tarea

### 1. `public/manifest.webmanifest` (nuevo)
Crear con exactamente esta forma (ajustá solo si algo choca con rutas reales):
```json
{
  "name": "Gastos Familiares",
  "short_name": "Gastos",
  "description": "Control de gastos de la familia",
  "start_url": "/",
  "scope": "/",
  "display": "standalone",
  "orientation": "portrait",
  "background_color": "#111827",
  "theme_color": "#1e3a5f",
  "lang": "es-AR",
  "icons": [
    { "src": "/icons/icon-192.png", "sizes": "192x192", "type": "image/png" },
    { "src": "/icons/icon-512.png", "sizes": "512x512", "type": "image/png" },
    { "src": "/icons/icon-maskable-512.png", "sizes": "512x512", "type": "image/png", "purpose": "maskable" }
  ],
  "share_target": {
    "action": "/share-target",
    "method": "POST",
    "enctype": "multipart/form-data",
    "params": {
      "files": [
        { "name": "files", "accept": ["application/pdf", "image/jpeg", "image/png", "image/webp"] }
      ]
    }
  }
}
```
- **Crítico:** el field name `files` y el path `/share-target` deben coincidir con
  `sw.js` (`data.getAll('files')`, `url.pathname === '/share-target'`). No los renombres.
- `accept` = el mismo set que aceptaba el legacy (`12_ShareTemp.gs`): PDF + JPEG/PNG/WebP.

### 2. Íconos PNG (nuevos, en `public/icons/`)
Generar desde el `icon.svg` existente (mismo "GF" blanco sobre `#1e3a5f`, esquinas
redondeadas):
- `icon-192.png` (192×192) y `icon-512.png` (512×512) — el "GF" ocupa ~65% del lienzo
  (como el SVG actual).
- `icon-maskable-512.png` (512×512) — **versión maskable**: fondo `#1e3a5f` a **sangre
  completa** (sin esquinas redondeadas, el SO las recorta) y el "GF" dentro del **safe zone
  del 80%** (más padding que la versión normal, para que no lo corte la máscara circular de
  Android).
- `apple-touch-icon.png` (180×180) — fondo `#1e3a5f` a sangre (iOS no respeta transparencia
  ni redondea bien; que se vea sólido).

### 3. `index.html` — links que faltan (iOS no lee todo del manifest)
Agregar en `<head>` (el `theme-color` y el `manifest` ya están):
```html
<link rel="apple-touch-icon" href="/icons/apple-touch-icon.png" />
<meta name="apple-mobile-web-app-capable" content="yes" />
<meta name="apple-mobile-web-app-status-bar-style" content="black-translucent" />
<meta name="apple-mobile-web-app-title" content="Gastos" />
```

## Aceptación
- `public/manifest.webmanifest` existe, válido, con `share_target` declarado; los íconos que
  referencia existen.
- En Chrome/Android (build de prod o `npm run preview`): la app **ofrece instalar**
  (Lighthouse → "Installable" en verde) y, una vez instalada, **aparece en el share sheet**;
  compartirle un PDF abre Comprobantes con el archivo precargado (vía el SW + IDB ya
  existentes).
- En iOS: "Agregar a inicio" usa el `apple-touch-icon` y abre en standalone (el share-to-app
  no aplica en iOS — queda el file picker in-app de Comprobantes como fallback, esperado).
- No se tocó `sw.js`, ni `src/`, ni el modelo.

## Notas
- Esto **no** agrega caching de red ni app-shell offline — eso es F9.50 (precache del shell +
  fallback de navegación), el salto de "instalable" a "abre sin red". Los **datos** ya
  funcionan offline por `persistentLocalCache` de Firestore.
- **iOS no soporta Web Share Target** (limitación de la plataforma, no del código). Dejar el
  picker in-app como fallback; no intentar emularlo.
- Verificación en dispositivo (Android install+share, iOS install) va al
  `docs/RUNBOOK-migracion-prod.md` como bloque "PWA (Etapa C)".
