# F8.1 — Menú "Clasificación y aprendizaje" + editor de Diccionario

## Objetivo

Exponer en **Perfil** un grupo nuevo, **admin-only**, que dé acceso a la base de
conocimiento del motor de matching (colecciones que hoy solo se editan por seed o
por los triggers `aprender*`). Esta fase entrega:

1. El **grupo + los 5 ítems del submenú** (IA completa a la vista).
2. La **ruta** y el **editor de Diccionario funcionando** (CRUD frontend puro).
3. Los otros 4 ítems (Destinos, Normalización, Subcategorías, Etiquetas) visibles
   con badge **"Pronto"** y sin navegación — se implementan en F8.2–F8.4.

**Fuera de alcance (NO tocar en este prompt):** callables, `firestore.rules`,
`functions/`. El editor de Diccionario escribe **client-side** porque las rules ya
lo permiten para `admin`. Deploy de esta fase: **`--only hosting`** (no hay backend).

## Contexto verificado (origin HEAD `df090fd`)

- `diccionario/{id}` rules: `read: esMiembro`; `create,update: esAdmin() || (esMiembro && origen=='Manual')`; `delete: esAdmin`. → **editable desde cliente por admin**.
- Shape del doc (espejo de `EntradaDict` en `src/datos/clasificador.ts`):
  `patron, tipoMatch('exact'|'contains'), descripcionLimpia, categoria, subcategoria, etiqueta, personaDefault, monedaDefault('ARS'|'USD'), bancoFiltro, tarjetaFiltro, confianza(number), activo(bool)` + metadata `origen, creadoPor, creadoEn`.
- El clasificador **renormaliza `patron` on-read** (ver comentario en `clasificar()`),
  así que las altas manuales **no requieren pre-normalizar**. Guardar el `patron` tal cual se tipea.
- Patrón de referencia para el editor: **`src/vistas/perfil/Categorias.tsx`** (mismo
  formato: editor de colección admin, variantes admin sin `where('activo','==',true)`,
  bottom-sheet/form del design-system). Espejar su estructura, no reinventar.
- Fuentes para dropdowns: categorías desde `useFamiliaConfig().config.categorias`;
  subcategorías/etiquetas desde `cargarSubcategoriasAdmin()`/`cargarEtiquetasAdmin()`
  en `src/datos/catalogos.ts`; personas desde `config.miembros`.

## Cambio 4 — Editor `src/vistas/perfil/Diccionario.tsx` (NUEVO)

Implementar espejando **`Categorias.tsx`** (misma cabecera con back, mismo bottom-sheet,
misma estética de filas). Spec funcional:

**Lectura**
- Cargar `diccionario` completo (incluye inactivos): `getDocs(collection(db,'diccionario'))`
  → mapear a `EntradaDict & { id: string }`. NO filtrar por `activo` (es editor admin).
- Ordenar por `patron` (locale `es-AR`, case-insensitive).

**Listado**
- Buscador arriba (filtra por `patron` y por `categoria`, substring case-insensitive).
- Cada fila muestra: `patron` (fuerte), línea secundaria `categoria › subcategoria`
  (o solo `categoria`), y a la derecha:
  - toggle `activo` (mismo switch que usa el resto de Perfil),
  - `pencil` (editar), `trash-2` (borrar, con confirm).
- Filas con `activo === false` atenuadas (opacity .55).
- Chip pequeño con `tipoMatch` ('exact' | 'contains') y, si `confianza != null`,
  `confianza` en % (read-only).

**Alta / edición (bottom-sheet)**
Campos:
- `patron` (text, requerido) — guardar tal cual (el clasificador renormaliza on-read).
- `tipoMatch` (segmented `contains` / `exact`, default `contains`).
- `categoria` (select desde `config.categorias.filter(c=>c.activo)`, requerido).
- `subcategoria` (select desde `cargarSubcategoriasAdmin()` filtrado por `categoriaPadre === categoria`, opcional → null).
- `etiqueta` (select desde `cargarEtiquetasAdmin()`, opcional → null).
- `personaDefault` (select desde `config.miembros` activos, opcional → null).
- `monedaDefault` (segmented ARS / USD / — , opcional → null).
- `bancoFiltro`, `tarjetaFiltro`: **no** exponer en v1 (dejar los valores existentes
  intactos al editar; en altas van `null`).

**Escritura (client-side, sin callable)**
- Alta: `addDoc(collection(db,'diccionario'), {...})` con:
  `activo: true, confianza: 0.9, origen: 'Manual', creadoPor: miMemberId(), creadoEn: serverTimestamp()`.
- Edición: `updateDoc(doc(db,'diccionario',id), {...campos})` — no tocar `confianza`,
  `origen`, `creadoEn`, `creadoPor` salvo que se editen explícitamente (no se editan en v1).
- Toggle activo: `updateDoc(..., { activo: !activo })`.
- Borrado: `deleteDoc(doc(db,'diccionario',id))` tras confirm.
- Tras cada escritura, refrescar la lista local (optimista o refetch — como Categorias.tsx).

**Notas**
- Sin real-time obligatorio: `getDocs` + refetch alcanza (igual criterio que Categorías).
- El cache del `DiccionarioContext` se hidrata al montar la app; **avisar en un
  helper text** que los cambios impactan el prellenado de sugerencias tras recargar
  (no invalidar el context acá — fuera de alcance).
