# F6.9.2 — La factura nace como obligación abierta (confirmadoPago:false)

## Problema (causa raíz de que el pago no reconcilie)
En `src/vistas/Comprobantes.tsx`, el preload del alta (rama 2/3) hace:
```ts
confirmadoPago: confirmadoPagoPorFecha(d.fecha),   // línea 170
```
`confirmadoPagoPorFecha` devuelve `fecha <= hoy`. Una **factura** con fecha de emisión pasada
(ej. colegio, emisión 05/06) nace con **`confirmadoPago: true`**. Pero F6.9 (`reconciliarPorPayee`)
solo considera obligaciones **abiertas** (`confirmadoPago === false`), así que **salta la factura**,
devuelve 0, y el pago del colegio cae a rama 3 → **crea un mov duplicado** en vez de reconciliar.

`confirmadoPago` está mezclando dos semánticas: "la fecha ya pasó" vs "la saldó un pago". Una factura
vieja **no está pagada** hasta que entra su transferencia.

Verificado en vivo (Run A limpio): factura_c → mov creado; pago del colegio → `→ rama 3` (no
`reconciliación por payee`) → duplicado.

## Decisión
Una **factura** (y cualquier comprobante que **no** sea pago) nace como **obligación abierta**:
`confirmadoPago: false`, sin importar su fecha. Solo un **pago** (transferencia / comprobante_pago)
puede nacer confirmado por fecha. Esto reusa el mismo criterio `esPago` que ya usa el server en
`matchComprobante`.

## Cambio (cliente, `src/vistas/Comprobantes.tsx`)
Antes del `const preloadBase = {`, definir:
```ts
const esPago = d.tipoDocumento === 'transferencia' || d.tipoDocumento === 'comprobante_pago';
```
Y cambiar la línea del preload:
```ts
// ANTES
confirmadoPago:      confirmadoPagoPorFecha(d.fecha),
// DESPUÉS — factura = obligación abierta; solo el pago se confirma por fecha
confirmadoPago:      esPago ? confirmadoPagoPorFecha(d.fecha) : false,
```
No tocar nada más. La propagación de `destinoCbu/Cuit/Alias` (líneas 173-176) ya está y es necesaria
para que F6.9 reconcilie.

## Efecto
- **Factura** → mov `confirmadoPago:false` → en ResumenMes aparece **por_confirmar** (obligación pendiente),
  y F6.9 **sí la encuentra** cuando llega el pago.
- **Pago del colegio** → F6.9 halla la obligación abierta por payee+monto → `rama 1 (reconciliación por payee)`
  → `confirmarRama1` setea `confirmadoPago:true` → pasa a **pagado**, sin duplicar.
- **Pago nuevo** (rama 3, sin obligación) → `esPago` true → `confirmadoPago` por fecha (reciente) → nace pagado. Correcto.
