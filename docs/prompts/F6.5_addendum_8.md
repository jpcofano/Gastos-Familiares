# F6.5 — Addendum 8 (quitar dedup del parseo de resumen)

> El parseo de un resumen de tarjeta descartaba líneas reales tratándolas como "duplicados".
> En la Visa Galicia 2026-04 (60 líneas, 2 personas) se perdió un cargo ANTHROPIC de U$S 5,00
> (línea #51), y el cuadre USD dio 44,99 calculado vs 49,99 del PDF. Fix: el resumen es una
> lista autoritativa; cada línea es una transacción real. Sin dedup en el parseo.

## Diagnóstico (verificado en el repo)
- `functions/src/index.ts:792` → `const movsBrutos = dedupMovimientos(parsed.movimientos);`
- `dedupMovimientos` (líneas 704-714) filtra por dos claves:
  ```ts
  const k1 = `${descripcionRaw.toUpperCase()}|${monto}`;
  const k2 = `${monto}|${moneda}|${fechaConsumo}`;
  if (seen.has(k1) || seen.has(k2)) return false;
  ```
- Caso real: línea 50 = `ANTHROPIC | U$S 5,00`. La línea #51 (otro `ANTHROPIC` U$S 5,00) generó el
  mismo `k1 = "ANTHROPIC|5"` → el filtro la borró. La 53 sobrevivió solo porque su descripción es
  `"ANTHROPIC in1TMrciB"` (otro k1). Resultado: descuadre exacto de U$S 5,00.
- `k2` es aún más riesgoso: descarta cualquier par con mismo monto + misma fecha + misma moneda
  **aunque sea de comercios distintos** (dos compras de $5.000 el mismo día → se pierde una).

## Decisión (A-over-B)
- **A (elegida):** el parseo del resumen preserva todas las líneas que emite el modelo. El **cuadre**
  es la única compuerta de integridad. Un resumen es una lista autoritativa: dos líneas idénticas son
  dos transacciones reales, no un duplicado.
- **B (rechazada):** deduplicar por descripción/monto/fecha. B solo puede borrar transacciones reales
  repetidas (cargos idénticos legítimos) y producir descuadres que parecen errores de parseo.
- El dedup tiene sentido en el flujo de **comprobantes** (no contar dos veces el mismo PDF, vía
  `hashPdf` — `matchComprobante` rama 0, líneas 218-235). Ese queda intacto. No aplica al resumen:
  si el modelo alucinara una línea de más, el cuadre lo caza igual (calculado > PDF).

## Cambios

### 1. Usar las líneas del modelo sin filtrar
`functions/src/index.ts:792`:
```diff
-      const movsBrutos = dedupMovimientos(parsed.movimientos);
+      const movsBrutos = parsed.movimientos;
```

### 2. Borrar la función muerta
`functions/src/index.ts:704-714`: eliminar `dedupMovimientos` (queda sin usar tras el cambio 1).
Verificar que no la llame nadie más (`grep dedupMovimientos functions/src/`).

## No tocar
- El dedup de **comprobantes** por `hashPdf` (`matchComprobante`, rama 0) — ese es correcto y de otro flujo.
- El cuadre (`calcularCuadre`) y el guard de `confirmarResumenTarjeta` — siguen siendo la compuerta de
  integridad; ahora con todas las líneas presentes deberían cerrar.
