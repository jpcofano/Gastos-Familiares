# F9.99.2 — HOTFIX: botón Chat por posición no hace nada (ReferenceError de scope)

> Prompt de feature para Code. Va a `docs/prompts/`. Prioridad: alta — el camino
> chat por posición (F9.99) está roto en producción; sectorial y agenda funcionan.

## Síntoma
En Tenencias, al expandir una posición y tocar "Chat", no pasa nada. El modal
nunca abre. Sectorial y agenda (ResearchTab) funcionan bien.

## Causa raíz (diagnosticada, verificada en el código)
`TenenciasTab` es un componente de nivel de módulo (línea ~912 de
`src/vistas/Patrimonio.tsx`), definido FUERA de `Patrimonio()` (línea ~2706).
Su firma no recibe `onAbrirChat`:

```ts
function TenenciasTab({ M, posiciones, manuales, fechaCorrida, analisisCache, configIA, onAnalizar }: {...})
```

Pero adentro (líneas ~1058-1063), el callback `onAbrirChat` que le pasa a
`AnalisisIASection` referencia directamente `setModalPromptChat` y
`setAnalisisCache` — que son setters de `useState` declarados DENTRO de
`Patrimonio()` (línea ~2749) y por lo tanto no existen en el scope de
`TenenciasTab`:

```ts
onAbrirChat={(ctx) => {
  const onDone = () => cargarAnalisisPosicion(c.ticker).then(a => {
    if (a) setAnalisisCache(prev => ({ ...prev, [c.ticker]: a }));   // fuera de scope
  });
  setModalPromptChat({ modo: 'posicion', ticker: c.ticker, contexto: ctx, onDone });  // fuera de scope
}}
```

En runtime, el tap lanza `ReferenceError: setModalPromptChat is not defined`
dentro del event handler → React lo traga → "no hace nada". (Verificable en la
consola del browser al tocar el botón.)

## Por qué ningún check lo detectó
`tsconfig.json` incluye solo `"src/**/*.ts"` — **los `.tsx` no se typechequean
nunca** (tampoco hay opción `jsx` configurada), y `vite build` usa esbuild, que
no tipa. El frontend entero está sin type checking.

## Fix (dos partes)

### Parte 1 — el bug (obligatoria)
Replicar el patrón que ya funciona en `ResearchTab` (que sí recibe `onAbrirChat`
como prop):

1. Agregar a la firma y tipos de `TenenciasTab`:
   `onAbrirChat: (ticker: string, contexto: Record<string, unknown>) => void;`
2. Reemplazar el callback de las líneas ~1058-1063 por:
   `onAbrirChat={(ctx) => onAbrirChat(c.ticker, ctx)}`
   (eliminar el `onDone` local: el refetch vive en el padre).
3. En `Patrimonio()`, en la invocación `<TenenciasTab ...>` (línea ~3072),
   pasar el prop con el mismo patrón del handler de ResearchTab (línea ~3170):
   ```tsx
   onAbrirChat={(ticker, contexto) => {
     const onDone = () => {
       cargarAnalisisPosicion(ticker).then(a => {
         if (a) setAnalisisCache(prev => ({ ...prev, [ticker]: a }));
       });
     };
     setModalPromptChat({ modo: 'posicion', ticker, contexto, onDone });
   }}
   ```

### Parte 2 — que no vuelva a pasar (recomendada, misma sesión)
1. En `tsconfig.json`: `"include": ["scripts/**/*.ts", "src/**/*.ts", "src/**/*.tsx"]`
   y agregar `"jsx": "react-jsx"` a compilerOptions.
2. Correr `npx tsc --noEmit` y arreglar los errores que aparezcan en `src/`
   (van a aparecer: hasta hoy los .tsx nunca se tiparon). Si son demasiados
   para una sesión, arreglar al menos todos los de `Patrimonio.tsx` y dejar
   el resto inventariado en el resumen de cierre.
3. Documentar en las reglas de deploy: `tsc --noEmit` manual antes de
   `firebase deploy` (mismo espíritu que compilar Functions a mano).

## Criterios de aceptación
1. Tenencias → expandir posición → Chat → abre el modal, genera el prompt.
2. Importar la respuesta actualiza el análisis de esa posición sin recargar.
3. Sectorial y agenda siguen funcionando (sin regresión).
4. `npx tsc --noEmit` corre sin errores en `Patrimonio.tsx`.
5. Push al cerrar.
