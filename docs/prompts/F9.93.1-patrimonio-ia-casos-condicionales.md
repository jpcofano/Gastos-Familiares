# F9.93.1 — Patrimonio: análisis IA con "qué haría en cada caso" (wording condicional)

## Contexto

Decisión del dueño: el análisis por posición debe dejar claro **qué se haría en
cada caso**. Se relaja la prohibición absoluta de lenguaje de acción: lo que
sigue prohibido es la **prescripción incondicional** ("vendé", "comprá ya",
"recomiendo salir"); lo que se habilita es la **guía condicional por caso**
("si pasa X → las opciones son A/B/C, con estos costos"). La decisión final
sigue siendo del dueño — el sistema mapea casos, no ordena.

Tocar solo: `buildPromptPosicion` / `buildPromptSectorial` en
`functions/src/index.ts`, y `AnalisisIASection` en la UI.

## 1) Prompt interno — modo posicion

Reemplazar el campo `senalesAVigilar` por uno más rico (o mantener ambos si
el shape lo permite sin alargar de más):

```
"queHariaEnCadaCaso": [
  {
    "caso": "condición observable y concreta (ej: 'si la revisión tarifaria sale desfavorable')",
    "acciones": ["2-3 opciones de acción posibles para ese caso"],
    "costo": "el trade-off principal de actuar (impositivo, upside resignado, timing)"
  }
]
```

- 2 a 4 casos por posición, ordenados del más probable al menos.
- Los casos deben ser **observables** (un dato, un evento, un precio), no
  vaguedades ("si empeora el clima").

Reglas actualizadas (reemplazan las actuales):
- PROHIBIDO: imperativos sin condición ("vendé", "comprá", "recomiendo
  salir/entrar"), precios objetivo presentados como certeza.
- PERMITIDO: condicionales con opciones ("si X, convendría evaluar A o B
  porque..."), siempre con el costo/trade-off explícito.
- La decisión es del titular: cada caso presenta opciones, nunca una única
  salida obligada.
- Se mantienen: español rioplatense, JSON estricto, no inventar (decir
  "sin información confiable" si aplica), web_search para lo actual.

## 2) Prompt interno — modo sectorial

Mismo criterio: por bloque (energía AR, soberano, cripto, tech global),
agregar `queHariaEnCadaCaso` a nivel sector (ej: "si el riesgo país comprime
por debajo de X → casos/opciones para la renta fija en cartera").

## 3) UI — `AnalisisIASection`

- Nueva sección "Qué haría en cada caso" (después de riesgos, antes de
  fuentes): cada caso como mini-card — condición en negrita, opciones como
  lista corta, costo en gris abajo.
- Si el JSON trae el campo viejo `senalesAVigilar` (análisis cacheados
  previos), renderizarlo como hasta ahora — retrocompatible, sin migración.

## 4) Documentar

- `docs/patrimonio/CLAUDE-PATRIMONIO.md`, sección Filosofía: actualizar de
  "nunca prescribir" a "nunca prescribir **incondicionalmente**; la guía por
  casos (condición → opciones → costo) es el formato permitido de acción".
- `docs/CLAUDE.md`: registrar F9.93.1.
