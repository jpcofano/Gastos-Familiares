# F9.64 — payee real en historial y landing + sacar nombre de archivo (punto 3 + payee del 4)

> Sigue a F9.63. Un solo archivo: `src/vistas/Comprobantes.tsx`. Extrae un helper y lo reutiliza en 4 lugares.
> El rediseño de layout del ShareLanding va aparte en F9.65 (este solo cambia la **fuente** del dato).

---

## Problema

En el historial (y en el landing PWA) se muestra `comercioRazonSocial`, que en transferencias es la **plataforma**
("Mercado Pago") y no aporta — el payee real ("Baggini Juan Francisco") está en `destinoNombre`. Además el **nombre
de archivo** ("IMG-…jpg") es el título de cada card del historial y es ruido.

La lógica correcta de payee (gateada por tipo) **ya existe** inline en `Comprobantes.tsx:220-223`; hay que extraerla
a un helper y reutilizarla.

---

## Cambios (todos en `src/vistas/Comprobantes.tsx`)

### 1. Extraer helper `payeeDeDatos`

Agregar a nivel módulo (por ejemplo justo antes de `function DatosResumen`). Usa el tipo `DatosExtraidos` ya
importado:

```ts
// Payee legible del comprobante: factura → emisor (comercioRazonSocial);
// transferencia/pago → destinatario (destinoNombre). Gateado por tipo, NO con ??:
// una billetera (Mercado Pago) llena comercioRazonSocial con su marca y taparía el destino.
function payeeDeDatos(d: DatosExtraidos): string | undefined {
  const esPagoDoc = d.tipoDocumento === 'transferencia' || d.tipoDocumento === 'comprobante_pago';
  return esPagoDoc
    ? (d.destinoNombre ?? d.comercioRazonSocial ?? undefined)
    : (d.comercioRazonSocial ?? d.destinoNombre ?? undefined);
}
```

### 2. Reutilizar el helper en la lógica inline

**BUSCAR:**
```ts
  const esPagoDoc = d.tipoDocumento === 'transferencia' || d.tipoDocumento === 'comprobante_pago';
  const descripcionCruda = esPagoDoc
    ? (d.destinoNombre   ?? d.comercioRazonSocial ?? undefined)
    : (d.comercioRazonSocial ?? d.destinoNombre   ?? undefined);
```
**REEMPLAZAR:**
```ts
  const descripcionCruda = payeeDeDatos(d);
```

### 3. Fila de detalle del historial (`DatosResumen`) — payee en vez de comercio

**BUSCAR:**
```tsx
      {d.comercioRazonSocial && <span>{d.comercioRazonSocial}</span>}
```
**REEMPLAZAR:**
```tsx
      {payeeDeDatos(d) && <span>{payeeDeDatos(d)}</span>}
```

### 4. Título de la card del historial — payee en vez de nombre de archivo

**BUSCAR:**
```tsx
        <span style={{ flex: 1, minWidth: 0, fontSize: 14, fontWeight: 600, whiteSpace: 'nowrap', overflow: 'hidden', textOverflow: 'ellipsis' }}>{comp.nombreArchivo}</span>
```
**REEMPLAZAR:**
```tsx
        <span style={{ flex: 1, minWidth: 0, fontSize: 14, fontWeight: 600, whiteSpace: 'nowrap', overflow: 'hidden', textOverflow: 'ellipsis' }}>{(comp.datosExtraidos && payeeDeDatos(comp.datosExtraidos)) || comp.nombreArchivo}</span>
```
> Fallback al nombre de archivo solo si el comprobante todavía no tiene `datosExtraidos` (estado subido/en proceso).
> Una vez extraído, muestra el payee ("Baggini Juan Francisco").

### 5. Landing PWA (FacturaLanding) — comercio = payee

**BUSCAR:**
```ts
          comercio: faseCompartido.comp.datosExtraidos.comercioRazonSocial,
```
**REEMPLAZAR:**
```ts
          comercio: payeeDeDatos(faseCompartido.comp.datosExtraidos) ?? null,
```
