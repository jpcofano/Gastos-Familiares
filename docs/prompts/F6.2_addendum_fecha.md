# F6.2 — Addendum: hardening de fecha (URGENTE)

> Hardening del prompt de extracción (functions/src/index.ts). Bug en producción de datos:
> con fechas sin año (ej. "13/jun"), el modelo tiró un año pasado arbitrario (2025).

## Causa
El SYSTEM_PROMPT pide `fecha = fecha de emisión` en ISO, pero:
1. NUNCA le dice al modelo qué día es hoy → no tiene referencia para inferir el año.
2. No hay regla para cuando el documento no muestra el año (comprobantes de Mercado Pago muestran
   "13/jun - 21:30" sin año).

## Fix
### a) Inyectar "hoy" en cada llamada
La function ya arma los mensajes — agregar la fecha actual al contexto que recibe el modelo (en el
SYSTEM_PROMPT armado en runtime, o en el bloque de texto del user message):
```
Hoy es {YYYY-MM-DD} (zona horaria America/Argentina/Buenos_Aires).
```
Usar `new Date()` de la function, formateado ISO en zona AR.

### b) Regla dura de fecha en el prompt (reemplaza la línea actual de `fecha`)
```
- fecha: ISO "YYYY-MM-DD" con año de 4 dígitos SIEMPRE. Es la fecha de emisión.
  Si el documento NO muestra el año explícitamente (ej. "13/jun"), asumí el año tal que la fecha sea
  la ocurrencia MÁS RECIENTE de ese día/mes que NO sea futura respecto de hoy. Ejemplos (hoy = 2026-06-16):
  "13/jun" → 2026-06-13; "20/dic" → 2025-12-20 (porque dic 2026 todavía no pasó).
  NUNCA elijas un año pasado arbitrario ni dejes un año por defecto.
```
Aplicar el mismo criterio de año a las fechas de `vencimientos[]` y al `YYYY-MM` del pseudo-número /
`periodoFacturado` cuando el año no esté explícito.

### c) Sanity check en la function (post-parseo)
- Si `fecha` parseada queda **en el futuro** respecto de hoy → es sospechosa. Loguear warning; opcional:
  marcar el comprobante para revisión (no bloqueante).

## Re-extracción de los 2 docs ya corruptos
El guard `estado !== 'subido'` bloquea re-extracción. Para corregir los que ya quedaron con año mal:
borrar el doc en el emulator UI y resubir (emulador volátil, sin drama). En prod sería un fix manual del
campo o un re-trigger controlado (fuera de alcance acá).
