# F9.41 — CRUD Tarjetas (editar cierre / vence / tipo) · cierra configs editables

> Reglas: doc-first · una PR. Reusa el patrón callable admin-only de F9.36/F9.37/F9.38.
> Habilita la edición de los campos que F9.35 agregó al schema pero dejó solo-lectura.
> Es la **última** de las 6 configs editables (cierra el hallazgo de paridad F9.32).

## Contexto
F9.35 agregó al modelo `FamiliaConfig.tarjetas[]` los campos `cierreDia?`, `venceDia?`,
`tipoTarjeta?: 'credito'|'debito'` (opcionales) y los **muestra** en Perfil › Tarjetas,
pero **no se editan** (en F9.35 se dejó así a propósito, para no romper la consistencia
"ninguna config edita" — que ya se levantó con F9.36–40). Ahora se cablea la edición.

## Tarea (`Perfil › Tarjetas`)
Callable admin-only (mismo patrón que `actualizarMediosPago`/`guardarTaxonomia`):
auth → `/autorizados` → validar → escribir en `config/familia`. Nunca write directo del
cliente (`config/{doc}` es `write:false` en Rules).

- **Editar** por tarjeta: `cierreDia` (1–31), `venceDia` (1–31), `tipoTarjeta`
  (credito/debito). Nombre/banco/red/terminación según lo que ya exista en el catálogo.
- **Alta / baja** de tarjeta del catálogo (si aplica al modelo actual; si las tarjetas se
  derivan de resúmenes, limitar a editar metadata, no crear/borrar — confirmar).
- Validación server-side: días en rango 1–31; `tipoTarjeta` en el enum; no fabricar
  fechas si quedan vacías (siguen mostrando "—", regla de F9.35).
- **Coherencia cuotas**: si `tipoTarjeta === 'debito'`, advertir/bloquear líneas en cuotas
  (débito no genera cuotas) — el warn que F9.35 dejó pasa a validación.

## UI
- Reusar el scaffold de edición de F9.36 (form controlado + dirty-state + guardar/cancelar).
- Solo admin; dependiente ve read-only.
- Sin `#fff` hardcodeado; tokens del kit; tema claro/oscuro.

## Aceptación
- Admin edita cierre/vence/tipo por tarjeta y persiste vía callable (no write directo).
- Días validados 1–31; tipo en enum; vacíos siguen mostrando "—" (no se inventan fechas).
- Débito no admite cuotas (coherencia validada server-side).
- Con esto, **las 6 configs del kit son editables** → cierra el hallazgo de paridad F9.32.
- Build verde.

  `npm run validate` + sign-off → habilita Etapa B (Firestore). Ver `CLAUDE.md`.
