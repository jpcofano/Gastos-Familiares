# F6.7 — Fix detector: SALDO ANTERIOR no es decisivo (falso positivo telco/servicios)

> El detector clasifica como `resumen` si hay un marcador decisivo. `MARCADORES_DECISIVOS` incluye
> `'SALDO ANTERIOR'`, pero eso aparece en facturas de telco/servicios (la factura de Personal dice
> "Saldo Anterior $138.805,49") → la rutearía a resúmenes de tarjeta. Falso positivo confirmado con
> ejemplo real.

## Diagnóstico (verificado en el repo + ejemplos)
- `functions/src/index.ts:1041-1043`:
  ```ts
  const MARCADORES_DECISIVOS = ['PAGO MINIMO', 'LIMITE DE COMPRA', 'LIMITE DE CREDITO', 'SALDO ANTERIOR'];
  ```
- `SALDO ANTERIOR` NO es card-only: "Mi Personal internet" (telco) tiene "Saldo Anterior" y además
  "Tu Resumen de Servicios". Con la regla actual `tieneDecisivo=true` → resumen. Mal.
- Los decisivos genuinamente solo-tarjeta son: `PAGO MINIMO`, `LIMITE DE COMPRA`, `LIMITE DE CREDITO`
  y el PAN enmascarado (`RE_PAN_ENMASCARADO`).

## Cambio
`functions/src/index.ts:1042`:
```diff
- const MARCADORES_DECISIVOS = ['PAGO MINIMO', 'LIMITE DE COMPRA', 'LIMITE DE CREDITO', 'SALDO ANTERIOR'];
+ const MARCADORES_DECISIVOS = ['PAGO MINIMO', 'LIMITE DE COMPRA', 'LIMITE DE CREDITO'];
```
(El PAN enmascarado ya entra por `RE_PAN_ENMASCARADO`, queda igual.)

Confirmar además que la regla de ruteo sea: **`resumen` solo si `tieneDecisivo`** (decisivo o PAN);
solo-genéricos (VENCIMIENTO/CIERRE/TOTAL A PAGAR/SALDO PENDIENTE) → `comprobante`. Así la **patente**
(que tiene VENCIMIENTO + CIERRE + "SALDO A PAGAR" pero ningún decisivo) cae en comprobante.

## Criterio de cierre (probado con los ejemplos reales)
- **Patente AGIP** (VENCIMIENTO, CIERRE, 1°/2° VTO, sin decisivo) → comprobante.
- **Personal** (Saldo Anterior + "Resumen de Servicios", sin decisivo real) → comprobante.
- **Aysa / Edenor / Metrogas / ABL / Expensas / Colegio** → comprobante.
- **Resumen real** (con PAGO MINIMO o PAN enmascarado) → resumen.

## Watch (menor)
`RE_PAN_ENMASCARADO = /\d{4}[\s*X]{4,10}\d{4}/` permite `\s` en el medio: un número de cuenta espaciado
largo (ej. "Cuenta 7 497 140 070") podría matchear de casualidad. Si aparece un falso positivo por
esto, restringir el set del medio a `[*X]` (sin `\s`) o exigir al menos un `*`/`X`.
