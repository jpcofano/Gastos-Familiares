# F9.87 — Navegación: Perfil desde el avatar · Patrimonio en el bottom-nav

> `AppShell`/`ShellFrame` + `BottomNav` + router. Referencia: kit `ui_kits/mobile/` (MobileShell `NAV` + `index.html`).

---

## Cambio

Hoy el bottom-nav es Inicio · Resumen · Cargar · **Perfil**. Se reorganiza:

- **Perfil** sale del bottom-nav y se abre **tocando el avatar** del header global (el avatar ya está en el header —
  F9.70; solo hay que hacer que navegue a `/perfil`).
- El **4º slot del bottom-nav** pasa a ser **Patrimonio** (ícono `landmark`, label "Patrimonio"), que navega a
  `/patrimonio` (la vista de F9.84).

## Gating (importante)

Patrimonio está gated a **jpcofano@gmail.com** (F9.84). Por lo tanto:

- **Solo el dueño** ve el 4º slot como "Patrimonio". Para cualquier otro usuario (otros admin, dependientes) ese
  slot **no debe mostrar Patrimonio**. Opciones (elegir la más simple en el código actual):
  - a) para no-dueño, el 4º slot queda como **Perfil** (y el avatar también abre Perfil), o
  - b) el bottom-nav muestra 3 ítems para no-dueño.
  Preferencia: **(a)** — 4º slot = Patrimonio para el dueño, Perfil para el resto; el avatar siempre abre Perfil.
- La ruta `/patrimonio` ya está protegida por el gate de F9.84 (no alcanza con ocultar el ícono).

## Detalles

- El avatar (inicial del miembro, círculo tinta) es un botón → `navigate('/perfil')`.
- Al estar en Perfil o en una sub-pantalla de Perfil, el bottom-nav no resalta ningún ítem (Perfil ya no es tab)
  — o resalta Patrimonio solo cuando la ruta es `/patrimonio`.
- Sin cambios en el resto de los tabs (Inicio/Resumen/Cargar).

## Test
1. Tocar el avatar abre Perfil desde cualquier pantalla.
2. Con `jpcofano@gmail.com`: el 4º ícono del bottom-nav es Patrimonio y abre `/patrimonio`.
3. Con otro usuario: el 4º ícono no es Patrimonio (queda Perfil) y `/patrimonio` sigue bloqueada por el gate.
