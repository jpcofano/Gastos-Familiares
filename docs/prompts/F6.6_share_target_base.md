# F6.6 — Compartir comprobante desde el celular (base)

> Modo B: base congelado. Cambios → addendum. PASO 1 puede dar vuelta el enfoque (ver gate de plataforma).
> Paridad con el sistema viejo (12_ShareTemp.gs: carpeta Drive temporal + token, workaround de Apps Script).

## Motivo
Compartir un PDF/imagen desde el teléfono (mail, WhatsApp, visor) directo a la app, sin abrirla a mano.
En Firebase NO se porta la carpeta Drive: el share llama a `subirComprobante` → Storage (F6.1) +
dedup SHA-256 + trigger onCreate (F6.2) hacen el resto. La infra pesada ya existe; esto es el "puente".

## PASO 1 — GATE (resolver ANTES de cualquier código)
1. **CRÍTICO — plataforma de los teléfonos de la familia (Android / iPhone / mixto).**
   La Web Share Target API (recibir archivos compartidos) anda en **Android** (Chrome/Edge, como PWA
   instalada). **iOS Safari NO la soporta** — no se puede registrar la app como destino de "Compartir".
   - Si es **Android** → seguir con PASO 2 (share_target nativo).
   - Si es **iPhone** → este enfoque NO sirve; el base se reemplaza por addendum con alternativa
     (input file con `capture` ya cubre subir desde el celular abriendo la app; o un Shortcut iOS que
     POSTea a una Cloud Function; o mantener un endpoint de upload tipo el viejo). **No implementar
     share-target en iOS.**
   - Si es **mixto** → share-target para Android + fallback de input file para iOS.
2. **¿La app ya es PWA?** (manifest + service worker registrado). Si no, es prerequisito del share-target
   y hay que pararlo primero. Reportar estado.

**Pará y reportá plataforma + estado PWA. Si es iOS-only, NO sigas con PASO 2.**

## PASO 2 — Implementación (path Android/PWA, con aprobación)
- En el manifest: `share_target` con `method: POST`, `enctype: multipart/form-data`, y `params.files`
  aceptando `application/pdf` + imágenes (mismo set que el viejo: pdf, jpeg, png, webp).
- Service worker: handler del POST de share → extrae el archivo del form-data → llama a `subirComprobante`
  (el mismo del flujo actual; dedup SHA-256 e idempotencia ya cubiertos).
- Validar tipo/tamaño igual que la subida normal (máx 10MB, tipos permitidos).
- Tras subir: el pipeline F6 (onCreate → extracción → match) corre solo. La UI `/comprobantes` lo muestra
  vía onSnapshot (F5.3) sin refetch.
- Fallback iOS (si plataforma mixta): asegurar que el input file con `accept` + `capture` cubre la subida
  manual desde el browser del celular.

## Cierre
1. Build + prueba en **Android real** (no emulador): instalar la PWA, compartir un PDF desde otra app
   → la app aparece en el menú de compartir → sube → se ve en `/comprobantes` → extrae. Verificar dedup
   (compartir el mismo dos veces no duplica).
2. `docs/CLAUDE.md`: cerrar F6.6 (mismo commit) con la plataforma soportada anotada.
3. Push: `F6.6 — share-target de comprobantes desde el celular`.
