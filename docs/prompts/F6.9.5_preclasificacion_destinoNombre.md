# F6.9.5 — Preclasificación por texto: usar `destinoNombre` en transferencias

## Problema
El preload clasifica con `clasificar(d.comercioRazonSocial)` (`Comprobantes.tsx:151`). Pero
`comercioRazonSocial` es la razón social del **emisor**, y en una **transferencia** el emisor es quien
**paga** (el "De"), no el payee — suele venir **null**. El payee ("Mingmingyu") vive en **`destinoNombre`**.

Resultado: para transferencias, `descripcionCruda = undefined` → el clasificador **ni se consulta** →
aunque el payee esté aprendido en el diccionario, **es inalcanzable**. De ahí "Mingmingyu está en el
diccionario y el form abre vacío".

(Esto es el carril **texto**, complementario a F6.9.4 que es el carril **destino/CBU**. Los dos hacen falta:
el destino cubre payees con CBU aprendido; el texto cubre el diccionario por nombre.)

## Cambio (cliente, `src/vistas/Comprobantes.tsx`)
BUSCAR:
```
  const descripcionCruda = d.comercioRazonSocial ?? undefined;
```
REEMPLAZAR:
```
  // payee del comprobante: factura → emisor (comercioRazonSocial); transferencia → destinatario (destinoNombre)
  const descripcionCruda = d.comercioRazonSocial ?? d.destinoNombre ?? undefined;
```

Con esto, para una transferencia el clasificador recibe el nombre del destinatario ("Mingmingyu"),
encuentra la entrada del diccionario, y prellena. Y la descripción del mov por defecto pasa a ser el payee,
consistente con lo que se aprende.
