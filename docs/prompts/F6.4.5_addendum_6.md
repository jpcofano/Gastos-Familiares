# F6.4.5 — Addendum 6 (guard de seed en aprendizaje)

> Modifica los triggers `aprenderMovimientoCreado` / `aprenderMovimientoActualizado` (functions/src/index.ts).
> Bug confirmado: al seedear con el emulador de Functions prendido, los ~1.137 movimientos del Excel
> disparan `onDocumentCreated` → `aprender()` corre 1.137 veces. NO es loop; es el trigger comiéndose el seed.

## Daño actual
1. **Traba** el emulador: cola de 1.137 ejecuciones de `aprender()`.
2. **Contamina el diccionario**: cada movimiento histórico infla `usoCount` y sube confianza (addendum 3)
   como si fuera una clasificación del usuario. El aprendizaje debe venir SOLO de acciones reales del
   usuario (alta/edición manual), NO del seed de data histórica ya categorizada.

## Fix — flag de seed + guard
1. **Marcar los movimientos seedeados** con un flag explícito, ej. `seedImport: true`, en el transformer
   del seed de `movimientos` (`scripts/seed/transformers/...`).
2. **Guard en AMBOS triggers**, al inicio (antes de `aprender()`):
   ```ts
   // aprenderMovimientoCreado
   const data = event.data?.data();
   if (data?.seedImport) return;                 // ← NUEVO: no aprender del seed
   if (!data?.categoria || !data?.subcategoria) return;
   await aprender(data)...
   ```
   ```ts
   // aprenderMovimientoActualizado
   const after = event.data?.after.data();
   if (after?.seedImport) return;                // ← NUEVO
   if (!after?.categoria || !after?.subcategoria) return;
   if (before?.categoria === after.categoria && before?.subcategoria === after.subcategoria) return;
   await aprender(after)...
   ```
   El flag es a prueba de olvidos (no depende de apagar el emulador de Functions al seedear).

## Limpiar la contaminación ya existente
El diccionario del emulador quedó contaminado por las corridas previas. Como el emulador es volátil y el
diccionario se re-seedea desde el Excel:
- `rmdir /s /q emulator-data` → reiniciar emuladores → `seed` (ahora con el flag puesto, el trigger no
  aprende del seed) → `validate`.
- Verificar que tras el seed el diccionario tenga **solo** las entradas del Excel (`usoCount` original),
  sin inflado por el trigger.

## Nota de diseño
`seedImport` también sirve a futuro: cualquier import masivo (migración a prod, re-seed) queda excluido del
aprendizaje por defecto. El aprendizaje es exclusivamente para movimientos nacidos de la app (manual,
comprobante, resumen) — esos NO llevan `seedImport`.

## Cierre
- `rmdir → seed → validate` verde, SIN cascada de `[aprender]` en los logs (el seed ya no dispara aprendizaje).
- Subir un comprobante / alta manual real → SÍ debe disparar `[aprender]` (una vez). Esa es la única fuente
  válida.
- Commit: `F6.4.5 — guard de seed en aprendizaje (seedImport)`.
