# F6.2.2 — Ampliar extracción de comprobantes (vencimientos, período, cliente)

> **Modo de trabajo**: investigá y reportá ANTES de escribir. Esperá aprobación explícita.
> Emulador-only. Patch del prompt de extracción de F6.2 + ampliación de tipo. NO toca el match (F6.3).

## Motivo

Las boletas de servicios argentinas traen **dos (o tres) vencimientos**: primer venc (monto base) y
segundo venc (con recargo, a veces +5–10%). Hoy el prompt no desambigua cuál monto es `montoTotal` ni
captura los vencimientos. Riesgo concreto sobre F6.3: si se extrae el monto del 2º venc pero el pago fue
del 1º, el recargo puede superar el ±5% del matcher → no matchea el movimiento → rama 2 (crear) en vez
de rama 1 (vincular) → **movimiento duplicado**. Este patch lo cierra capturando todos los vencimientos.

## Decisiones (cerradas)

- **`montoTotal` = primer vencimiento / pronto pago** (monto base). NUNCA el segundo. Es lo que
  normalmente se paga y lo que la UI muestra. Para comprobantes sin vencimientos (ticket, factura simple),
  `montoTotal` = total del documento, como hasta ahora.
- **`vencimientos[]`**: array `0..n`, ordenado, `[0]` = primer venc (su `monto` == `montoTotal`).
  Para docs sin vencimientos → array vacío `[]`.
- **`periodoFacturado`**: "YYYY-MM" si se puede normalizar, sino el texto crudo. Pasa a ser la fuente
  preferida del pseudo-número (`YYYY-MM-<slug>`), con la `fecha` de emisión como fallback.
- **`numeroCliente`**: nro de cliente / cuenta / suministro (servicios). Identificador estable del
  servicio entre meses; null si no aplica.
- **Mínimo viable** sin cambios: `tipoDocumento + moneda + numeroOperacion`. Los campos nuevos son todos
  opcionales/nullable → no agregan requisito duro, no rompen docs no-servicio.
