# F9.39 — Config: TC manual (entrada de respaldo a /tcDiario)

> Reglas: doc-first · una PR. Reusa patrón F9.36. La pantalla **Tipo de cambio** pasa a
> **escribir** `tcDiario` (hoy solo lee). Es el **respaldo manual** del refresco automático
> F9.30.

## Contexto
F9.30 agenda una Cloud Function que actualiza `tcDiario` desde dolarapi (MEP). Pero si la
API falla, o se quiere forzar/corregir un valor, hace falta poder **cargar el TC del día a
mano**. La fuente sigue siendo única (`tcDiario`); esta pantalla solo agrega escritura.

## Tarea (`Perfil › Tipo de cambio`)
- **Mostrar**: TC actual (último `tcDiario`), su fecha y **origen** (`dolarapi-bolsa` vs
  `manual`), + histórico reciente (ya se lee, `cargarTCReciente()`).
- **Editar / cargar**: valor del TC para una fecha (default hoy, zona AR). Upsert en
  `tcDiario/{YYYY-MM-DD}` con `origen:'manual'` y `actualizadoEn`. **Mismo doc/shape** que
  escribe la function F9.30 — no una colección paralela.
- **Solo admin**. Avisar si se sobrescribe un valor existente (manual pisa automático para
  ese día; el cron del día siguiente vuelve a poner el automático).

## Integridad
- No crear otra fuente de TC: escribe el **mismo** `tcDiario` que lee `tcParaFecha` y que
  alimenta todos los montos derivados. Un solo lugar de verdad.
- `tipoTarjeta`/cuotas y los equivalentes ARS/USD se recalculan solos al cambiar el TC de
  una fecha (son derivados).

## Aceptación
- Admin carga/corrige el TC de un día; se persiste en `tcDiario` con `origen:'manual'`.
- `tcParaFecha` lo toma sin cambios; los montos del Dashboard/Resumen reflejan el nuevo TC
  por fecha.
- No hay fuente de TC duplicada. Build verde.
