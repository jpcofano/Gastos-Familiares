# F9.92.1 — Resumen: "pendientes del mes" a check verde en 0 + card Hoy con total por banco

> `src/vistas/Resumen.tsx` → `PorDiaSeccion` (fila "Revisar pendientes del mes" y card "Hoy").

---

## §1 · "Revisar pendientes del mes" → check verde cuando `porRevisar === 0`

Hoy la fila siempre muestra `alert-circle` + badge numérico, incluso con `porRevisar === 0` (badge "0", que no
comunica nada). Cambiar a estado binario:

```tsx
<Card variant="flat" padding="var(--space-3)" onClick={onIrAGastos} style={{ display: 'flex', alignItems: 'center', gap: 10, cursor: 'pointer' }}>
  <Icon name={porRevisar === 0 ? 'check-circle-2' : 'alert-circle'} size={17} color={porRevisar === 0 ? 'var(--gf-income)' : 'var(--gf-out)'} />
  <span style={{ flex: 1, fontSize: 14, fontWeight: 600, color: porRevisar === 0 ? 'var(--gf-income)' : 'var(--color-text)' }}>
    {porRevisar === 0 ? 'Al día con los gastos fijos' : 'Revisar pendientes del mes'}
  </span>
  {porRevisar > 0 && (
    <span style={{ minWidth: 22, height: 22, borderRadius: 999, background: 'var(--gf-out)', color: '#fff', fontSize: 12, fontWeight: 700, display: 'inline-flex', alignItems: 'center', justifyContent: 'center', padding: '0 6px' }}>{porRevisar}</span>
  )}
</Card>
```

- Ícono: si no existe `check-circle-2` en el set de iconos del proyecto, usar el que ya se use para "pagado"/"al día"
  en otro lado (buscar `name="check"` con fondo circular, patrón usado en `HoyCard`/checklist).
- Mantiene el `onClick` a la solapa Gastos Fijos en ambos estados (no hay razón para sacarlo).

## §2 · Card "Hoy" — agregar total por banco

Hoy la card muestra: header con total del día + "a pagar", y debajo la lista ítem por ítem. Falta el desglose por
banco (mismo patrón visual que ya existe en "Gastos por día": píldora gris con `BankLogo` + nombre + monto).

Insertar el desglose **entre el header y la lista de ítems**, agregando por banco los `hoyItems` que ya tienen banco
conocido (pagados/conciliados hoy vía `ci.matches[0]?.banco`). Los pendientes sin match no tienen banco todavía — no
entran en este desglose (quedan representados en "a pagar" del header).

```tsx
const hoyPorBanco = new Map<string, number>();
for (const ci of hoyItems) {
  const banco = ci.matches[0]?.banco;
  if (!banco) continue;
  const monto = ci.matches.reduce((s, m) => s + Math.abs(m.monto), 0); // ARS eq del match real
  hoyPorBanco.set(banco, (hoyPorBanco.get(banco) ?? 0) + monto);
}
const hoyBancos = [...hoyPorBanco.entries()].sort((a, b) => b[1] - a[1]);
```

```tsx
{hoyBancos.length > 0 && (
  <div style={{ display: 'flex', flexWrap: 'wrap', gap: 5, marginBottom: 10 }}>
    {hoyBancos.map(([b, v]) => {
      const info = bancoDeNombre(b, config?.bancos);
      return (
        <span key={b} style={{ display: 'inline-flex', alignItems: 'center', gap: 5, fontSize: 11, fontWeight: 600, color: 'var(--color-text-strong)', background: 'var(--gf-gray-100)', borderRadius: 999, padding: '3px 8px 3px 4px' }}>
          <BankLogo id={info?.id ?? b} nombre={info?.nombre ?? b} color={info?.color ?? colorMedio(b, config?.bancos) ?? colorHash(b)} dominio={info?.dominio} size={17} radius={999} />
          {b} · {fmtArs(v)}
        </span>
      );
    })}
  </div>
)}
```

- Mismo componente `BankLogo`/estilo píldora que en "Gastos por día" — no inventar un patrón nuevo.
- Si `hoyItems.length === 0` (nada que pagar hoy) no se muestra nada de esto — sigue el mensaje actual "Nada que pagar hoy.".
- El total general (header) y la lista por ítem ya están bien — no tocar esa parte.

## Test
1. Mes con al menos un ítem accionable pendiente: fila roja con badge numérico + tap → Gastos Fijos (sin cambios).
2. Mes con `porRevisar === 0`: fila pasa a ícono + texto verde, sin badge numérico.
3. Día con ≥2 pagos conciliados en bancos distintos: aparecen las píldoras de banco con el monto correcto sumado por banco, antes de la lista de ítems.
4. Día sin nada pagado todavía (todo "a pagar"): no aparece ninguna píldora de banco (no hay banco conocido aún).
