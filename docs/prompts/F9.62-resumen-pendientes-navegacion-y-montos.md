## Problemas

1. **"Revisar pendientes del mes"** (fila con badge de cantidad) no es clickeable — debería llevar a la solapa
   **Gastos Fijos**. Y el número cuenta de más: hoy incluye `por_confirmar` (ítems ya cargados, falta confirmar);
   debe contar solo lo **sin cargar** (sin movimiento).
2. **Card PENDIENTE** (solapa Gastos Fijos) da **$0** aunque haya ítems no pagados: hoy suma solo `montoEsperado`
   de ítems no cubiertos **con monto definido**; los ítems sin `montoEsperado` (o cargados sin confirmar) no aportan.
   Debe sumar **todo lo no pagado**.

### 4. Reconteo de `porRevisar` — solo lo SIN CARGAR

**BUSCAR:**
```tsx
  const porRevisar = checklist.filter(c => ACCIONABLE.includes(c.estado)).length;
```
**REEMPLAZAR:**
```tsx
  // F9.62 — "revisar" cuenta solo lo SIN CARGAR (sin movimiento asociado). por_confirmar
  // tiene match (cargado, falta confirmar) y NO entra en este conteo.
  const porRevisar = checklist.filter(c => c.matches.length === 0 && ACCIONABLE.includes(c.estado)).length;
```
> Efecto: quedan solo `pendiente`/`vencido`/`no_registrado` (los que tienen `matches.length === 0`); `por_confirmar`
> sale. El número baja.

### 5. Card PENDIENTE — suma todo lo no pagado

**BUSCAR:**
```tsx
  const pendiente = checklist.filter(c => !cubierto(c.estado) && c.item.montoEsperado != null).reduce((s, c) => s + (c.item.montoEsperado ?? 0), 0);
```
**REEMPLAZAR:**
```tsx
  // F9.62 — "pendiente" suma TODO lo no pagado: si el ítem tiene un movimiento sin confirmar
  // (por_confirmar/parcial) usa el monto REAL del movimiento; si no, el montoEsperado. Ítems
  // sin monto ni match no aportan (no hay número que sumar).
  const pendiente = checklist
    .filter(c => !cubierto(c.estado))
    .reduce((s, c) => {
      const noConfirmado = c.estado === 'por_confirmar' || c.estado === 'parcial';
      const montoReal = c.matches.reduce((a, m) => a + Math.abs(m.monto), 0);
      return s + (noConfirmado ? montoReal : (c.item.montoEsperado ?? 0));
    }, 0);
```

## Notas

- **Mezcla de monedas:** el card muestra `fmtArs(pendiente)` y la suma es **cruda** (ARS + USD sin convertir),
  igual que el código original. Si hay ítems USD, se mezclan. No se corrige acá para no ampliar el alcance; si querés
  conversión ARS-eq, va en un addendum aparte (existe `c.tc` en la sección de KPIs).
- `cubierto(estado)` = `pagado` || `automatico` (checklist.ts). "No cubierto" incluye `por_confirmar`, `parcial`,
  `pendiente`, `vencido`, `no_registrado`, `programado`.
