# F6.9.3 — Extracción: la factura debe poblar `destinoCuit` (= emisor)

## Problema (causa raíz de que el pago no reconcilie)
En el prompt del extractor (`functions/src/index.ts`), el campo `destinoCuit` tiene la regla:
*"el destino es la parte 'Para'/'Destinatario'/'Beneficiario'; NUNCA la parte 'De'/'Origen'"*.
En una **factura**, el comercio (emisor) es la parte "De" y no hay un "Para", así que el modelo deja
**`destinoCuit: null`** y mete el CUIT del comercio solo en `cuit`. La aclaración existente
("si es factura de comercio, el CUIT del receptor es el comercio") pierde contra el "NUNCA De/Origen".

Consecuencia: el mov de la factura **no tiene ningún identificador de payee** (`destinoCuit`/`destinoCbu`
nulos → ni siquiera dispara `[aprenderDestino]`). El pago sí trae payee, no comparten nada,
`reconciliarPorPayee` da 0 → rama 3 → **duplica**. Verificado en vivo (Run A limpio): factura → solo
`[aprender]` (texto), nunca `[aprenderDestino]`; pago → `[aprenderDestino] (cbu)`; pago cae en `→ rama 3`.

## Decisión
El "payee" es a **quién se le paga**. En una **factura/comprobante de comercio** ese payee **es el emisor**
(el comercio que la emitió y al que vas a pagar). Por lo tanto, para facturas:
`destinoCuit` = el mismo CUIT del emisor, `destinoNombre` = `comercioRazonSocial`, `destinoCbu` = CBU de
cobro **si figura**. Para transferencias/pagos no cambia nada (sigue siendo la parte "Para").

## Cambios (prompt del extractor, `functions/src/index.ts`)

### BUSCAR
```
- destinoCuit: CUIT/CUIL del destinatario (solo dígitos, 11 dígitos, sin guiones). REGLA CRÍTICA: el destino es la parte "Para"/"Destinatario"/"Beneficiario"; NUNCA la parte "De"/"Origen" (que es el CUIT del titular que paga). Si el comprobante es QR/factura de comercio, el CUIT del receptor es el comercio. null si no hay receptor claro.
```
### REEMPLAZAR
```
- destinoCuit: CUIT/CUIL del PAYEE (a quién se le paga), solo dígitos, 11 dígitos, sin guiones.
  · TRANSFERENCIA/PAGO: es la parte "Para"/"Destinatario"/"Beneficiario"; NUNCA la parte "De"/"Origen" (el titular que paga).
  · FACTURA / COMPROBANTE DE COMERCIO: NO hay parte "Para". El payee del pago futuro ES EL EMISOR (el comercio que emite la factura). Por lo tanto destinoCuit = el MISMO CUIT del emisor (igual valor que el campo cuit, pero solo dígitos sin guiones).
  null si no hay payee claro.
```

### BUSCAR
```
- destinoNombre: nombre o razón social del destinatario/beneficiario. null si no aplica.
```
### REEMPLAZAR
```
- destinoNombre: nombre o razón social del PAYEE. Para transferencia/pago = el destinatario; para factura = el emisor (mismo que comercioRazonSocial). null si no aplica.
```

No tocar `destinoCbu` (ya dice "Para factura/servicio = CBU de cobro si figura") ni `cuit`.
