## §1 · Toggle Lista / Dona / Treemap (icono + label)

El control debe renderizar **3 segmentos con icono Y texto** (no icono solo — así, si un icono falta, el label igual
se ve). Idéntico a `seg()` del kit:

```tsx
const seg = (id, icon, label) => {
  const on = tipo === id;
  return (
    <button key={id} onClick={() => set(id)} aria-label={label} title={label}
      style={{ display:'inline-flex', alignItems:'center', gap:5, padding:'5px 9px', borderRadius:8,
        border:'none', cursor:'pointer', fontSize:11.5, fontWeight: on?700:600,
        background: on?'var(--color-surface)':'transparent', color: on?'var(--color-text)':'var(--color-text-sec)',
        boxShadow: on?'var(--shadow-sm)':'none' }}>
      <Icon name={icon} size={14} />{label}
    </button>
  );
};
// contenedor: display:flex; gap:2; background:var(--gf-gray-100); borderRadius:10; padding:3
// seg('lista','list','Lista')  seg('dona','chart-pie','Dona')  seg('treemap','layout-grid','Treemap')
```

- Verificá que los iconos `list`, `chart-pie`, `layout-grid` existan en el icon set de la app; si falta alguno,
  agregalo (o usá el equivalente lucide). El label siempre presente.
- El header de la card es `flex; align-items:center; justify-content:space-between` → título+"este mes" a la izq,
  el toggle a la der. Sin overflow: a 360px entra (fontSize 11.5, padding chico).

## §2 · No pisar el total con la barra

El kit **no** tiene un total flotante dentro de la card "Por categoría" (los totales del período viven en el header
del Dashboard). En el vivo el `$8.258` está mal posicionado y la barra apilada lo tapa. Opciones (en orden de
preferencia):

1. **Quitarlo** de esa card y confiar en el total del período de arriba (paridad exacta con el kit), **o**
2. Si se quiere conservar, moverlo a una **línea propia** debajo del header (no en la fila de la barra) para que no
   se superponga.

La barra apilada va **full-width** (`height:12, borderRadius:999, overflow:hidden, marginBottom:16`), con un segmento
por categoría (`width: c.pct%`).

## §3 · Top subcategorías → estilo lista de categorías

Reescribir cada fila al patrón del kit: **punto (9×9, color de la subcat) + nombre a la izquierda**, monto + chip %
empujados a la derecha (`margin-left:auto`), y **barra full-width debajo** escalada al **máximo** de la lista.

```tsx
{subcats.map((s) => (
  <div key={s.nombre}>
    <div style={{ display:'flex', alignItems:'center', gap:8, fontSize:13, marginBottom:4 }}>
      <span style={{ width:9, height:9, borderRadius:3, background:s.color, flexShrink:0 }} />
      <span style={{ fontWeight:600, whiteSpace:'nowrap', overflow:'hidden', textOverflow:'ellipsis' }}>{s.nombre}</span>
      <span style={{ marginLeft:'auto', display:'inline-flex', alignItems:'center', gap:8 }}>
        <span style={{ fontWeight:700, fontVariantNumeric:'tabular-nums' }}>{curBig(s.valor, cur)}</span>
        <span style={{ fontSize:11, fontWeight:700, color:'var(--gf-gray-400)', background:'var(--gf-gray-100)',
          borderRadius:999, padding:'1px 7px', fontVariantNumeric:'tabular-nums' }}>{s.pct}%</span>
      </span>
    </div>
    <div style={{ height:6, background:'var(--gf-gray-100)', borderRadius:3, overflow:'hidden' }}>
      <div style={{ height:'100%', width:`${Math.max((s.valor/maxSub)*100, 4)}%`, background:s.color, borderRadius:3 }} />
    </div>
  </div>
))}
```

- `maxSub = Math.max(...subcats.map(s => s.valor))` → la barra más grande llega al 100%; piso 4%.
