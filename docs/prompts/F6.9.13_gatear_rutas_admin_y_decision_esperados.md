# F6.9.13 — Cerrar rutas de admin al dependiente + decisión de `itemsEsperados`

Dos cosas que cierran el frente "dependiente fuera de las superficies de admin":

1. **Decisión `itemsEsperados` (doc-only):** los esperados son **familiares** (sin dato
   por-persona; solo admin los crea/edita). El read se deja **intencionalmente abierto a
   `esMiembro()`** — no es dato sensible y el `ItemsEsperadosProvider` los lee app-wide para
   mostrar info de ítems en el flujo de comprobantes del dependiente; scopearlo a admin le
   rompería esa vista con permission-denied. Create/update ya es admin-only (`firestore.rules:77`).
   No hay cambio de código; se documenta y se cierra la decisión abierta.

2. **Fix real:** las rutas `/resumen` y `/config-esperados` están montadas **sin gatear**
   (`AppShell.tsx:77-78`; solo el nav está detrás de `esAdmin`). Un dependiente llega por URL
   y ve el ResumenMes con totales de **toda la familia** — justo lo que no debe ver. Gatear
   esas rutas a admin + catch-all que redirige.

## Cambio 1 — `src/AppShell.tsx`: gatear rutas de admin

`Navigate` y `esAdmin` ya están disponibles (import línea 1; `esAdmin` línea 44).

BUSCAR:
```
          <Routes>
            <Route path="/" element={<Dashboard />} />
            <Route path="/resumen" element={<Resumen />} />
            <Route path="/config-esperados" element={<ConfigEsperados />} />
            <Route path="/comprobantes" element={<Comprobantes />} />
            <Route path="/tarjetas" element={<Navigate to="/comprobantes" replace />} />
          </Routes>
```
REEMPLAZAR:
```
          <Routes>
            <Route path="/" element={<Dashboard />} />
            {esAdmin && <Route path="/resumen" element={<Resumen />} />}
            {esAdmin && <Route path="/config-esperados" element={<ConfigEsperados />} />}
            <Route path="/comprobantes" element={<Comprobantes />} />
            <Route path="/tarjetas" element={<Navigate to="/comprobantes" replace />} />
            <Route path="*" element={<Navigate to="/" replace />} />
          </Routes>
```

Por qué el catch-all: con las rutas de admin removidas para el dependiente, `/resumen` y
`/config-esperados` no matchean ⇒ el `*` lo manda a `/` (Dashboard) en vez de pantalla en
blanco. Para admin, las rutas existen y el `*` no se toca.
