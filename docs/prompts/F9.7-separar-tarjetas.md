# F9.7 — Separar Tarjetas: visor vs config

> Reglas: doc-first · una PR por fase · sin tocar Firestore/Functions · paridad
> funcional con la legacy · rumbo visual moderno 2026 (ink/esmeralda) · móvil primero.

Hoy "Tarjetas" mezcla dos cosas. Separalas en dos vistas distintas:

### 1. `/tarjetas` → VISOR de resúmenes (solo lectura)
- Es el componente actual de resúmenes de tarjeta (caras de tarjeta ink + total del
  resumen + estado pagado/parcial/pendiente + nº de consumos).
- Se llega desde **Resumen** (botón "Tarjetas" en el header del Resumen). Back vuelve
  a Resumen. No editable.

### 2. Perfil › "Tarjetas" → CONFIG propia
- Pantalla de configuración (mismo patrón que "Medios de pago"): lista de tus tarjetas
  con `banco · red`, `•••• term`, **cierre día N · vence día N**, titular; chevron para
  editar; botón **"Agregar tarjeta"**.
- Define los ciclos de cierre/vencimiento que alimentan los resúmenes del visor.
- Nota al pie: "Los resúmenes se ven en Resumen › Tarjetas".

### Routing / roles
- Quitar el acceso al visor desde Perfil; Perfil › Tarjetas abre **config**.
- El visor queda accesible solo desde Resumen (ruta `/tarjetas`).
- Config es admin (igual que el resto de "Configuración familiar"); el visor lo ven
  todos los roles.
- Modelo de datos: los campos de **config** (cierreDia, venceDia, titular, red, term)
  son la fuente; el visor consume esos ciclos + los consumos del período.

PR visual + routing (sin tocar Firestore/Functions).
