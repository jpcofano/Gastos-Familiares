## §1 · Tipo `Posicion` (espejo del contrato, NO reusar tipos de gastos)

```ts
type Posicion = {
  cuenta: string;            // "Balanz 402665"
  ticker: string;            // "TRAN"
  nombre: string;            // "Transener"
  tipo: 'accion'|'bono'|'on'|'cedear'|'fci'|'cripto'|'cash';
  sector: string;            // "Energía AR" | "Cripto" | "Renta fija AR" | "Bancos AR" | "Global"
  pais: 'AR'|'global';
  monedaOrigen: 'ARS'|'USD';
  valorUsd: number;          // la APP lo calcula al cargar: ARS → valorOrigen / tcDiario[fecha]
  revisar?: boolean;
};
```
`valorUsd` lo calcula la app vía `tcDiario` (fuente única del TC, ya existente). No online, no por-bróker.

## §2 · Motor de métricas (determinístico, front por ahora)

Idéntico al kit (`usePatMetrics`): total, peso por sector/tipo/país, top-1/3/5, HHI, sector top, % cripto, % RV.

- **HHI** = Σ (peso_i)². **Top-1 "nombre más grande"** = mayor tenencia individual **excluyendo cripto**
  (la cripto se mide como CLASE con banda propia, no como nombre único — no doble-contar).
- **% en renta variable** (acción + cedear + cripto) es **informativo, SIN semáforo** (la postura busca RV alta).

## §3 · Semáforos — bandas (UCITS 5/10/40, HHI DOJ)

| Métrica | 🟢 Verde | 🟡 Amarillo | 🔴 Rojo |
|---|---|---|---|
| Nombre único (excl. cripto) | ≤ 5% | 5–10% | > 10% |
| Sector | < 25% | 25–40% | > 40% |
| País único | < 40% | 40–60% | > 60% |
| Cripto (clase) | < 10% | 10–20% | > 20% |
| HHI | < 0,15 | 0,15–0,25 | > 0,25 |

Colores: verde `--gf-emerald`, ámbar `--gf-out`, rojo `--gf-expense`.

## §4 · Solapas (4, segmentado arriba)

1. **Resumen (foto)** — valor total USD + eq ARS (al `tcDiario` de la fecha del snapshot) + composición por sector
   (barra apilada + lista con %). *El kit trae 3 variantes A/B/C del Resumen; implementá la que el PM elija
   (ver nota abajo) — no las tres.*
2. **Tenencias** — todas las posiciones agrupadas por sector (header con total+% del sector), cada papel su línea
   (chip ticker + nombre + tipo·país·moneda + valorUsd + % del total). Sin agrupar dentro del sector.
3. **Riesgo** — las 5 métricas de §3 como tarjetas con semáforo (valor grande + banda impresa) + línea top-3/top-5
   + la nota de "% RV informativo, sin semáforo".
4. **Plan** — "idea madre" (mantener % RV, cambiar composición) + las 4 palancas de rebalanceo del doc (recortar
   nombres grandes → separar reguladas/productoras → redesplegar a otros sectores AR → sumar RV global) + disclaimer
   (no asesoría matriculada; ganadores en USD = costo impositivo, gradual; parte en cuenta conjunta).
